## 🔴 ОБЯЗАТЕЛЬНЫЙ ЧЕК-ЛИСТ (ПЕРЕД ЛЮБЫМИ ИЗМЕНЕНИЯМИ)

### Шаг 1: Бэкап (CRITICAL)

```markdown
[ ] 1.1: Проверить статус Git
       Команда: git status
       Ожидание: Нет незакоммиченных важных изменений

[ ] 1.2: Создать точку отката
       Команда: git add . && git commit -m "Backup: Перед [причина]"
       Ожидание: Коммит создан

[ ] 1.3: Сохранить текущее состояние
       Команда: .\scripts\auto-archive.ps1 -Reason "Before [action]"
       Ожидание: Архив создан
```

**Почему это важно:**  
Без бэкапа → потеря данных → 4-5 дня работы насмарку (см. [`ERROR_LOG.md`](./ERROR_LOG.md), ERR-001)

---

### Шаг 2: Анализ Рисков

```markdown
[ ] 2.1: Определить тип действия
       □ Создание файла
       □ Изменение файла
       □ Удаление файла
       □ Изменение структуры
       □ Другое: _______

[ ] 2.2: Оценить уровень риска (по матрице QWEN.md)
       □ 🟢 Зелёный (автономно)
       □ 🟡 Жёлтый (по контексту)
       □ 🔴 Красный (спросить подтверждения)

[ ] 2.3: Проверить анти-паттерны
       Команда: .\scripts\check-anti-patterns.ps1 -BeforeAction
       Ожидание: Нет нарушений
```

**Связанный файл:** [`ANTI_PATTERNS.md`](./ANTI_PATTERNS.md)

---

### Шаг 3: Проверка на Дубликаты

```markdown
[ ] 3.1: Поиск существующих решений
       Команда: .\scripts\check-duplicates.ps1 -Query "[тема]"
       Ожидание: Нет дубликатов ИЛИ найден существующий файл

[ ] 3.2: Если найден похожий файл → Дополнить его
       [ ] Найдено: _______
       [ ] Решение: Дополнить / Создать новый
```

---

### Шаг 4: Проверка Encoding (для новых файлов)

```markdown
[ ] 4.1: Установить кодировку UTF-8 без BOM
       [ ] Для .ps1 файлов
       [ ] Для .bat файлов
       [ ] Для .json файлов
       [ ] Для .md файлов с русским текстом

[ ] 4.2: Проверить после создания
       Команда: .\scripts\check-encoding.ps1 -Path "[файл]"
       Ожидание: UTF-8 без BOM
```

**Почему это важно:**  
Неправильная encoding → 5+ проваленных сессий (см. [`ERROR_LOG.md`](./ERROR_LOG.md), ERR-004)

---

### Шаг 5: Проверка 3-Уровневого Процесса

```markdown
[ ] 5.1: Создан черновик в _drafts/?
       [ ] ДА / [ ] НЕТ (если НЕТ → создать)

[ ] 5.2: Протестировано в _TEST_ENV/?
       [ ] ДА / [ ] НЕТ (если НЕТ → запустить тесты)

[ ] 5.3: Проверены ссылки?
       Команда: .\scripts\check-rule-links.ps1 -Path "_drafts/[файл]"
       Ожидание: 100% ссылок работают

[ ] 5.4: Получено подтверждение?
       [ ] ДА (пользователь подтвердил)
       [ ] НЕТ (спросить перед продолжением)
```

**Связанный файл:** [`.qwen/QWEN.md`](./.qwen/QWEN.md) — раздел "3-уровневый процесс"

## 📋 СПЕЦИФИЧЕСКИЕ ЧЕК-ЛИСТЫ

### Для Создания Файлов

```markdown
[ ] 1. Бэкап создан (Шаг 1)
[ ] 2. Проверены дубликаты (Шаг 3)
[ ] 3. Создано в _drafts/ (Шаг 5.1)
[ ] 4. Encoding: UTF-8 без BOM (Шаг 4)
[ ] 5. Заполнены мета-поля:
    [ ] Версия: 0.1 (черновик)
    [ ] Дата: YYYY-MM-DD
    [ ] Статус: WIP
[ ] 6. Протестировано в _TEST_ENV/ (Шаг 5.2)
[ ] 7. Ссылки проверены (Шаг 5.3)
[ ] 8. Подтверждение получено (Шаг 5.4)
```

---

### Для Изменения Файлов

```markdown
[ ] 1. Бэкап создан (Шаг 1)
[ ] 2. Определён уровень риска (Шаг 2.2)
[ ] 3. Если 🔴 Красный → получено подтверждение
[ ] 4. Изменения внесены
[ ] 5. Проверены ссылки после изменений
[ ] 6. Закоммичено с понятным сообщением
```

---

### Для Удаления Файлов

```markdown
[ ] 1. Бэкап создан (Шаг 1)
[ ] 2. Проверено: файл не используется в ссылках
       Команда: .\scripts\check-file-usage.ps1 -Path "[файл]"
[ ] 3. Dry-run удаление
       Команда: .\scripts\safe-delete.ps1 -Path "[файл]" -DryRun
[ ] 4. Просмотр отчёта
[ ] 5. Подтверждение получено
[ ] 6. Фактическое удаление
       Команда: .\scripts\safe-delete.ps1 -Path "[файл]" -AutoConfirm
```

---

### Для Коммитов

```markdown
[ ] 1. git status → все ли файлы добавлены
[ ] 2. Проверены изменения (git diff --staged)
[ ] 3. Сообщение коммита понятное:
    [ ] Формат: Type: Описание
    [ ] Детали (если нужно)
[ ] 4. Нет чувствительных данных (пароли, ключи)
[ ] 5. Коммит создан
```

**Формат сообщения:**
```
Add: Система работы над ошибками

- ERROR_LOG.md — журнал ошибок
- ANTI_PATTERNS.md — запрещённые паттерны
- PRE_ACTION_CHECKLIST.md — чек-лист
```

---

### Для Сборки EXE

```markdown
[ ] 1. Проверка окружения
       Команда: .\scripts\check-environment.ps1
       Ожидание: Unity установлен, лицензия активна

[ ] 2. Очистка логов

       [ ] Запускается (если возможно)

[ ] 5. Тесты проходят
       Команда: .\scripts\test-all.ps1
       Ожидание: 100%
```

**Связанный файл:** [`.qwen/QWEN.md`](./.qwen/QWEN.md) — раздел "UNITY BUILD CHECKLIST"
