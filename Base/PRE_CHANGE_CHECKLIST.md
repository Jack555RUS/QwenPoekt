## 🎯 НАЗНАЧЕНИЕ

Этот чек-лист **обязателен к выполнению** перед любыми изменениями в Ядре системы (Base/).

**Применяется для:**
- ✅ Создания новых файлов в Base/
- ✅ Изменения структуры проекта
- ✅ Удаления файлов/папок
- ✅ Массовых операций (>10 файлов)
- ✅ Изменения конфигурации ИИ

**Правило:** Не выполнил чек-лист → **НЕ НАЧИНАЙ ДЕЙСТВИЯ!**

## 🔴 ОБЯЗАТЕЛЬНЫЙ ЧЕК-ЛИСТ (8 шагов)

### **Шаг 1: 7 вопросов перед действием**

```markdown
[ ] 1. ЗАЧЕМ это нужно?
       Обоснование: _______
       
[ ] 2. ЧТО изменится?
       Файлы: _______
       
[ ] 3. ГДЕ будет размещено?
       Путь: _______
       
[ ] 4. КАК это реализовать?
       План: _______
       
[ ] 5. ЧТО ЗАТРОНЕТ?
       Зависимости: _______
       
[ ] 6. ЧТО СЛОМАЕТ?
       Риски: _______
       
[ ] 7. КАК ОТКАТИТЬ?
       План отката: _______
```

**Почему это важно:** Без ответа на эти вопросы действия бесцельны и опасны!

---

### **Шаг 2: Бэкап (CRITICAL)**

```markdown
[ ] 2.1: Проверить статус Git
       Команда: `git status`
       Ожидание: Нет незакоммиченных важных изменений

[ ] 2.2: Создать точку отката
       Команда: `git add . && git commit -m "Backup: Перед [причина]"`
       Ожидание: Коммит создан

[ ] 2.3: Сохранить текущее состояние
       Команда: `.\scripts\auto-archive.ps1 -Reason "Before [action]"`
       Ожидание: Архив создан
```

**Почему это важно:** Без бэкапа → потеря данных → 4-5 дня работы насмарку (см. [`ERROR_LOG.md`](./ERROR_LOG.md), ERR-001)

---

### **Шаг 3: Оценка рисков**

```markdown
[ ] 3.1: Определить тип действия
       □ Создание файла
       □ Изменение файла
       □ Удаление файла
       □ Изменение структуры
       □ Другое: _______

[ ] 3.2: Оценить уровень риска (по матрице QWEN.md)
       □ 🟢 Зелёный (автономно)
       □ 🟡 Жёлтый (по контексту)
       □ 🔴 Красный (спросить подтверждения)

[ ] 3.3: Проверить анти-паттерны
       Команда: `.\scripts\check-anti-patterns.ps1 -BeforeAction`
       Ожидание: Нет нарушений
```

**Связанный файл:** [`ANTI_PATTERNS.md`](./ANTI_PATTERNS.md)

### **Шаг 4: Проверка на дубликаты**

```markdown
[ ] 4.1: Поиск существующих решений
       Команда: `.\scripts\check-duplicates.ps1 -Query "[тема]"`
       Ожидание: Нет дубликатов ИЛИ найден существующий файл

[ ] 4.2: Если найден похожий файл → Дополнить его
       [ ] Найдено: _______
       [ ] Решение: Дополнить / Создать новый
```

**Связанный файл:** [`DEDUP_GUIDE.md`](./02-Areas/Documentation/DEDUP_GUIDE.md)

### **Шаг 5: Тестирование в _TEST_ENV/**

```markdown
[ ] 5.1: Создать тестовую среду

       Ожидание: _TEST_ENV/ создана

### **Шаг 6: Проверка именования**

```markdown
[ ] 6.1: Проверить имя файла
       [ ] Латиница (без кириллицы)
       [ ] Нижний регистр
       [ ] Дефисы (kebab-case)
       [ ] Длина ≤50 символов

[ ] 6.2: Проверить Encoding
       [ ] UTF-8 без BOM
       [ ] Команда: `chcp 65001`

[ ] 6.3: Добавить front matter (для .md)
       ```markdown
       ---
       title: Название на русском
       version: 1.0
       date: 2026-03-03
       status: active | draft | archived
       tags: [тег1, тег2]
       ---
       ```
```

**Связанный файл:** [`03-Resources/Knowledge/01_RULES/file_naming_rule.md`](./03-Resources/Knowledge/01_RULES/file_naming_rule.md)
