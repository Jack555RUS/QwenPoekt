## 📋 **ПРОБЛЕМА:**

**ИИ допускает ошибки в PowerShell скриптах:**

### **Тип 1: Экранирование `$`**

```powershell
# ❌ НЕПРАВИЛЬНО (внутри @" ... "@)
pwsh -Command @"
Set-Location /c/workspace; `$testScript
"@

# ✅ ПРАВИЛЬНО
pwsh -Command @"
Set-Location /c/workspace; $testScript
"@
```

**Правило:** Внутри `@" ... "@` экранирование НЕ нужно!

---

### **Тип 2: Кавычки в строках**

```powershell
# ❌ НЕПРАВИЛЬНО
$dockerVolume = "${basePath}:/c/workspace"

# ✅ ПРАВИЛЬНО
$dockerVolume = "$basePath`:/c/workspace"
```

**Правило:** Использовать обратные кавычки для экранирования `:`

---

### **Тип 3: Heredoc (@")**

```powershell
# ❌ НЕПРАВИЛЬНО (смешение кавычек)
$testScript = @"
Write-Host "Тест 1: Проверка..."
`$folders = @('.qwen', '01-Projects')
"@

# ✅ ПРАВИЛЬНО
$testScript = @"
Write-Host 'Тест 1: Проверка...'
`$folders = @('.qwen', '01-Projects')
"@
```

**Правило:** Использовать одинарные кавычки внутри heredoc

---

### **Тип 4: Replace с `\`**

```powershell
# ❌ НЕПРАВИЛЬНО
$dockerVolume = "${basePath}:/c/workspace".Replace('\', '/')

# ✅ ПРАВИЛЬНО
$dockerVolume = ($basePath + ":/c/workspace").Replace('\', '/')
```

**Правило:** Избегать `\` в строковых литералах

---

## 🛠️ **РЕШЕНИЯ:**

### **Решение 1: Использовать простые строки**

```powershell
# Вместо сложного экранирования
$dockerVolume = "d:/qwenpoekt/base:/c/workspace"
```

---

### **Решение 2: Тестировать перед использованием**

```powershell
# Запустить в PowerShell ISE или VS Code
# Проверить синтаксис
Invoke-ScriptAnalyzer -Path script.ps1
```

---

### **Решение 3: Использовать шаблоны**

```powershell
# Создать шаблон безопасного скрипта
$template = @"
param(`$Path)
Write-Host "Путь: `$Path"
"@
```

---

### **Решение 4: Запросить проверку у пользователя**

```
⚠️ ВОЗМОЖНА ОШИБКА СИНТАКСИСА

Пожалуйста, проверьте:
1. Экранирование `$ (не нужно внутри @" ... "@)
2. Кавычки (использовать ' внутри heredoc)
3. Пути (использовать / вместо \)

Продолжить? (Y/N)
```

## 📚 **ПРАВИЛА ДЛЯ ИИ:**

### **Правило 1: Heredoc (@")**

```
Внутри @" ... "@:
- ✅ Использовать одинарные кавычки '...'
- ✅ Не экранировать `$ (просто `$variable)
- ✅ Использовать / для путей
```

---

### **Правило 2: Строковые литералы**

```
В обычных строках:
- ✅ Использовать двойные кавычки "..."
- ✅ Экранировать `$ как ``$
- ✅ Использовать / вместо \
```

---

### **Правило 3: Replace**

```
Для замены путей:
- ✅ ($path).Replace('\', '/')
- ✅ Избегать `\` в строках
```

---

### **Правило 4: Проверка**

```
Перед созданием скрипта:
- ✅ Предложить пользователю проверить синтаксис
- ✅ Использовать Invoke-ScriptAnalyzer
- ✅ Тестировать в PowerShell ISE
```
