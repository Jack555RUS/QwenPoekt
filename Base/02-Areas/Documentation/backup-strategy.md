# Стратегия резервного копирования

**Версия:** 1.0  
**Дата:** 2026-03-02  
**Проект:** QwenPoekt\Base

---

## 🎯 ЦЕЛЬ

Обеспечение полной сохранности данных проекта с использованием принципа **3-2-1**.

**Почему это важно:**

- ✅ **Защита от сбоя** — восстановление после потери данных
- ✅ **Защита от ошибок** — откат неудачных изменений
- ✅ **История изменений** — возможность вернуться к предыдущей версии
- ✅ **Спокойствие** — уверенность в сохранности работы

---

## 📊 ПРИНЦИП 3-2-1

### Классическая формула

```
3 копии данных
2 разных типа носителей
1 копия вне площадки (офлайн/облако)
```

### Адаптация для QwenPoekt\Base

| Копия | Тип | Расположение | Частота |
|-------|-----|--------------|---------|
| **Копия 1** | Локальная (основная) | `D:\QwenPoekt\Base` | Постоянно |
| **Копия 2** | Локальная (бэкап) | `D:\QwenPoekt\_BACKUP` | Перед операциями |
| **Копия 3** | Удалённая (Git) | GitHub | По запросу/коммиту |

## 🗂️ УРОВНИ БЭКАПА

### Уровень 1: Предоперационный бэкап

**Когда:** Перед удалением/перемещением/массовым изменением

**Скрипт:** [`pre-operation-backup.ps1`](PRE_OPERATION_BACKUP_GUIDE.md)

**Команда:**
```powershell
.\scripts\pre-operation-backup.ps1 -OperationType "Delete_OldProject"
```

**Что делает:**
- Копирует всю папку Base в `_BACKUP/`
- Проверяет целостность (количество файлов)
- Записывает в `OPERATION_LOG.md`
- Возвращает путь к бэкапу

**Время:** ~2-5 минут

**Хранение:** 45 дней → анализ → удаление

---

### Уровень 2: Ежедневный Git коммит

**Когда:** Ежедневно в 18:00

**Скрипт:** [`auto-commit-daily.ps1`](../../03-Resources/PowerShell/auto-commit-daily.ps1)

**Команда:**
```powershell
.\scripts\auto-commit-daily.ps1
```

**Что делает:**
- Git add всех изменений
- Git commit с сообщением
- (Опционально) Git push на GitHub

**Что коммитить:**
- ✅ `KNOWLEDGE_BASE/` (база знаний)
- ✅ `03-Resources/PowerShell/` (скрипты)
- ✅ `02-Areas/Documentation/` (документация)
- ✅ `_templates/` (шаблоны)
- ✅ `reports/` (отчёты)
- ❌ `PROJECTS/` (Unity проекты, локально)
- ❌ `OLD/` (библиотека наработок)
- ❌ `RELEASE/` (готовые проекты)

**Время:** ~1 минута

---

### Уровень 3: Тестовая копия

**Когда:** Перед тестированием скриптов

**Скрипт:** [`create-test-env.ps1`](TEST_ENV_GUIDE.md)

**Команда:**
```powershell
.\scripts\create-test-env.ps1
```

**Что делает:**
- Создаёт `_TEST_ENV/Base` (облегчённая копия)
- Исключает тяжёлые папки (PROJECTS, OLD, RELEASE, BOOK)
- Создаёт тестовые скрипты (`test-*.ps1`)
- Инициализирует логи тестов

**Время:** ~1 минута

**Хранение:** До очистки (`cleanup-test-env.ps1`)

## 📅 РАСПИСАНИЕ БЭКАПА

### Автоматическое (Task Scheduler)

**Скрипт:** [`schedule-backup-tasks.ps1`](../../03-Resources/PowerShell/schedule-backup-tasks.ps1)

**Задачи:**

| Задача | Время | Скрипт |
|--------|-------|--------|
| **Ежедневный коммит** | 18:00 ежедневно | `auto-commit-daily.ps1` |
| **Еженедельный аудит** | 09:00 воскресенье | `weekly-dedup-audit.ps1` |
| **Ежемесячная очистка** | 1-е число, 10:00 | `old-backup-cleanup.ps1` (будет) |

**Установка:**
```powershell
.\scripts\schedule-backup-tasks.ps1
```

---

### Ручное (по необходимости)

| Событие | Скрипт |
|---------|--------|
| **Перед удалением** | `pre-operation-backup.ps1` |
| **Перед тестами** | `create-test-env.ps1` |
| **По завершении задачи** | `auto-commit-daily.ps1` |
| **По запросу (GitHub)** | `github-backup.ps1` |
| **Завершение сессии** | `end-session.ps1` |

**Команда завершения сессии:**
```powershell
# Автоматически (по команде ИИ)

.\scripts\end-session.ps1
```

**Что делает:**
- Git коммит всех изменений
- Обновление OPERATION_LOG.md
- Создание отчёта о сессии
- Вывод статистики

**Время:** ~30 секунд

## 🛠️ ВОССТАНОВЛЕНИЕ ИЗ БЭКАПА

### Сценарий 1: Случайное удаление файла

**Восстановление из предоперационного бэкапа:**

```powershell
# 1. Найти бэкап
ls D:\QwenPoekt\_BACKUP | Sort-Object CreationTime -Descending

# 2. Скопировать файл
Copy-Item "D:\QwenPoekt\_BACKUP\2026-03-02_03-30_Delete_OldProject\KNOWLEDGE_BASE\file.md" `
          "D:\QwenPoekt\Base\KNOWLEDGE_BASE\file.md"
```

---

### Сценарий 2: Неудачное массовое изменение

**Восстановление из Git:**

```powershell
# 1. Найти коммит
git log --oneline

# 2. Восстановить файл
git restore KNOWLEDGE_BASE/file.md

# 3. Или восстановить всю папку
git restore KNOWLEDGE_BASE/
```

---

### Сценарий 3: Полный сбой системы

**Восстановление из GitHub:**

```powershell
# 1. Клонировать репозиторий
git clone https://github.com/Jackal/QwenPoekt.git D:\QwenPoekt\Restore

# 2. Проверить файлы
ls D:\QwenPoekt\Restore\Base

# 3. Скопировать важное
Copy-Item "D:\QwenPoekt\Restore\Base\KNOWLEDGE_BASE" `
          "D:\QwenPoekt\Base\KNOWLEDGE_BASE" -Recurse
```

## ⚠️ ПРЕДУПРЕЖДЕНИЯ

### Не полагаться на один тип бэкапа!

**❌ ОШИБКА:** Только Git

> Git не спасёт, если случайно удалил файл и закоммитил удаление!

**✅ РЕШЕНИЕ:** Git + предоперационный бэкап

---

### Не хранить бэкапы вечно!

**❌ ОШИБКА:** Бэкапы занимают 100 GB

> Через год бэкапы могут занять больше места, чем оригинал!

**✅ РЕШЕНИЕ:** 45 дней → анализ → 60 дней → удаление

---

### Не тестировать на оригинале!

**❌ ОШИБКА:** Тесты на Base → повреждение данных

> Тестирование скриптов на оригинале может привести к потере данных!

**✅ РЕШЕНИЕ:** Использовать `_TEST_ENV/`
