# Руководство по дедупликации Базы Знаний

**Версия:** 1.0  
**Дата:** 2026-03-02  
**Проект:** QwenPoekt\Base

---

## 🎯 ЦЕЛЬ

Автоматическое обнаружение и удаление дубликатов файлов для поддержания чистоты Базы Знаний.

**Почему это важно:**

- ✅ **Экономия места** — не хранятся повторы
- ✅ **Экономия времени** — не тратится на просмотр одного и того же
- ✅ **Чистота поиска** — результаты не дублируются
- ✅ **Воспроизводимость** — точное знание состава данных

---

## 📊 ТИПЫ ДУБЛИКАТОВ

### Тип 1: Точные дубликаты (Exact duplicates)

**Описание:** Идентичные копии файлов с одинаковым содержимым.

**Примеры:**
- `file_naming_rule.md` и `file_naming_rule_copy.md`
- Один и тот же PDF, скачанный дважды
- Копия скрипта в другой папке

**Обнаружение:** SHA-256 хэш (100% точность)

**Действие:** ❌ Удалить копию, оставить оригинал

---

### Тип 2: Почти одинаковые (Near duplicates)

**Описание:** Незначительно отличающиеся версии одного содержания.

**Примеры:**
- Статья в Markdown и HTML
- Скрипт с незначительными изменениями
- Файл с другим кодированием

**Обнаружение:** grep_search + ручной анализ

**Действие:** ⚠️ Проверить вручную (объединить или связать)

---

### Тип 3: Семантические дубликаты (Semantic duplicates)

**Описание:** Разные по форме, но одинаковые по смыслу.

**Примеры:**
- Две статьи об одном правиле
- Два решения одной ошибки
- Две версии одного руководства

**Обнаружение:** Qwen Code (ИИ-аналитик)

**Действие:** 🔄 Объединить / связать / отклонить

## 🛠️ ИНСТРУМЕНТЫ

### 1. generate-file-hashes.ps1

**Назначение:** Создание индекса хэшей (SHA-256)

**Использование:**

```powershell
# Создать индекс
.\scripts\generate-file-hashes.ps1

# Пересоздать индекс
.\scripts\generate-file-hashes.ps1 -Force
```

**Что делает:**
- Сканирует все файлы в Base/
- Исключает: PROJECTS, OLD, RELEASE, _BACKUP, _TEST_ENV, .git
- Вычисляет SHA-256 для каждого файла
- Сохраняет в `meta/file_hashes.json`
- Находит группы дубликатов

**Время:** ~1 минута (80 000 файлов)

---

### 2. check-hash-duplicates.ps1

**Назначение:** Проверка дубликатов по индексу

**Использование:**

```powershell
# Проверка (без удаления)
.\scripts\check-hash-duplicates.ps1

# Подробный вывод
.\scripts\check-hash-duplicates.ps1 -Verbose

# Автоматическое удаление (ОПАСНО!)
.\scripts\check-hash-duplicates.ps1 -AutoFix
```

**Что делает:**
- Загружает индекс хэшей
- Находит группы файлов с одинаковым хэшем
- Выводит список дубликатов
- Генерирует отчёт (`reports/DEDUP_AUDIT_REPORT.md`)
- (Опционально) Удаляет дубликаты

**Время:** ~30 секунд

---

### 3. weekly-dedup-audit.ps1

**Назначение:** Еженедельный автоматический аудит

**Использование:**

```powershell
# Запустить вручную
.\scripts\weekly-dedup-audit.ps1

# Автоматически (Task Scheduler)
.\scripts\schedule-backup-tasks.ps1
```

**Что делает:**
- Запускает `generate-file-hashes.ps1`
- Запускает `check-hash-duplicates.ps1`
- Создаёт отчёт (`reports/DEDUP_WEEKLY_AUDIT.md`)
- Записывает в `OPERATION_LOG.md`

**Время:** ~2 минуты

## 📋 ПРОЦЕСС ДЕДУПЛИКАЦИИ

### Этап 1: Генерация индекса

```powershell
# 1. Создать индекс
.\scripts\generate-file-hashes.ps1

# Вывод:
# === ГЕНЕРАЦИЯ ИНДЕКСА ХЭШЕЙ ===
# Файлов для хэширования: 82735
# Хэшей вычислено: 82730
# Дубликатов: 5 групп
```

**Что проверяет:**
- ✅ Все ли файлы просканированы
- ✅ Нет ли ошибок хэширования
- ✅ Сколько групп дубликатов найдено

---

### Этап 2: Анализ дубликатов

```powershell
# 2. Проверить дубликаты
.\scripts\check-hash-duplicates.ps1

# Вывод:
# === ПРОВЕРКА ДУБЛИКАТОВ ПО ХЭШАМ ===
# Найдено групп дубликатов: 5
#
# Группа #1
# Хэш: a1b2c3d4e5f6...
#   1. ✅ KNOWLEDGE_BASE/01_RULES/file_naming_rule.md
#   2. ✅ KNOWLEDGE_BASE/01_RULES/file_naming_rule_copy.md
```

**Что проверяет:**
- ✅ Тип дубликата (точный/почти/семантический)
- ✅ Существует ли файл
- ✅ Какой файл оставить (оригинал vs копия)

---

### Этап 3: Принятие решения

**Матрица решений:**

| Тип дубликата | Пример | Действие |
|---------------|--------|----------|
| **Точный файл-копия** | `file.md` и `file_copy.md` | Удалить копию |
| **Разные форматы** | `file.md` и `file.pdf` | ✅ Оставить оба |
| **Новая версия** | `file_v1.md` и `file_v2.md` | ✅ Оставить обе, указать связь |
| **Перефразированное** | Два файла об одном | ⚠️ Объединить или связать |

---

### Этап 4: Удаление (опционально)

**Вариант A: Автоматическое (AutoFix)**

```powershell
# ⚠️ ВНИМАНИЕ: Удалит все дубликаты, кроме первого!
.\scripts\check-hash-duplicates.ps1 -AutoFix
```

**Вариант B: Ручное (рекомендуется)**

```powershell
# 1. Открыть отчёт
code reports\DEDUP_AUDIT_REPORT.md

# 2. Проверить каждый дубликат

# 3. Удалить лишние
Remove-Item "KNOWLEDGE_BASE/01_RULES/file_naming_rule_copy.md"
```

## ⚠️ ПРЕДУПРЕЖДЕНИЯ

### AutoFix (автоматическое удаление)

**⚠️ ОПАСНОСТЬ:**

```powershell
# Удалит ВСЕ дубликаты, кроме первого в группе!
.\scripts\check-hash-duplicates.ps1 -AutoFix
```

**Риски:**
- ❌ Может удалить важную версию
- ❌ Не различает оригинал/копию
- ❌ Нельзя отменить

**Рекомендация:** Используйте ручное удаление!

---

### Исключения (не удалять)

**Не удалять:**

| Файл | Причина |
|------|---------|
| `*_v1.md`, `*_v2.md` | Разные версии (история) |
| `*.md`, `*.pdf` | Разные форматы (польза) |
| `draft_*.md`, `final_*.md` | Черновик и финал |

**Правило:**

> Если файлы имеют разное назначение → оставить оба!
