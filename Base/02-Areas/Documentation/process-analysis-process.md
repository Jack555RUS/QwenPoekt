## 🎯 НАЗНАЧЕНИЕ

Этот файл описывает **полный цикл обработки информации** в проекте QwenPoekt\Base.

**Используйте для:**
- Анализа новых материалов (книги, статьи, документация)
- Извлечения ключевых идей
- Оценки ценности (1-5)
- Создания метаданных
- Добавления в Базу Знаний

---

## 1️⃣ CAPTURE (Захват)

## 2️⃣ ORGANIZE (Организация)

### **Категоризация (PARA):**

| Категория | Критерий | Куда |
|-----------|----------|------|
| **Projects** | Есть дедлайн | `01-Projects/[название]/` |
| **Areas** | Постоянная ответственность | `02-Areas/[название]/` |
| **Resources** | Полезно для будущего | `03-Resources/Knowledge/` |
| **Archives** | Завершено/неактивно | `04-Archives/[год]/` |

---

### **Шаблон организации:**

```markdown
---
title: [Название]
version: 1.0
date: 2026-03-03
status: active | draft | archived
category: project | area | resource | archive
tags: [тег1, тег2, тег3]
related: [связанные файлы]
---

# 📖 [НАЗВАНИЕ]

**Категория:** Project / Area / Resource / Archive

**Связанные файлы:**
- [`файл1.md`](путь)
- [`файл2.md`](путь)

---

## 📋 СОДЕРЖАНИЕ

[Структурированное содержание]

---

**Следующий шаг:** Перейти к DISTILL
```

## 3️⃣ DISTILL (Дистилляция)

### **Оценка ценности (1-5):**

| Оценка | Критерий | Действие |
|--------|----------|----------|
| **⭐⭐⭐⭐⭐ (5)** | Критично важно | Сохранить в Ядро |
| **⭐⭐⭐⭐ (4)** | Очень полезно | Сохранить в Resources |
| **⭐⭐⭐ (3)** | Полезно | Сохранить с пометкой |
| **⭐⭐ (2)** | Малоценно | Архивировать |
| **⭐ (1)** | Бесполезно | Удалить |

---

### **Метод дистилляции:**

**Уровень 1: Выделение ключевых идей**
```markdown
## 🔑 КЛЮЧЕВЫЕ ИДЕИ

1. [Идея 1] — 1-2 предложения
2. [Идея 2] — 1-2 предложения
3. [Идея 3] — 1-2 предложения
```

**Уровень 2: Прогрессивное суммаризирование**
```markdown
## 📝 КРАТКОЕ ИЗЛОЖЕНИЕ

[Сжатое изложение в 3-5 предложениях]

## 💎 САМАЯ СУТЬ

[Одно предложение — главная мысль]
```

---

### **Шаблон дистилляции:**

```markdown
---
distilled: 2026-03-03
value: 5 | 4 | 3 | 2 | 1
---

## 🔑 КЛЮЧЕВЫЕ ИДЕИ

1. ...
2. ...
3. ...

## 📝 КРАТКОЕ ИЗЛОЖЕНИЕ

...

## 💎 САМАЯ СУТЬ

...

---

**Следующий шаг:** Перейти к EXPRESS
```

## 4️⃣ EXPRESS (Выражение)

### **Формы выражения:**

| Форма | Пример | Куда |
|-------|--------|------|
| **Письмо** | Статья, правило | `02-Areas/Documentation/` |
| **Код** | Скрипт, функция | `scripts/` или `03-Resources/PowerShell/` |
| **Обучение** | Объяснение кому-то | Сессия с ИИ |
| **Внедрение** | Изменение в проекте | Git коммит |

---

### **Шаблон выражения:**

```markdown
---
expressed: 2026-03-03
form: article | code | teaching | implementation
---

## 🎯 РЕЗУЛЬТАТ

[Что создано/изменено]

## 📊 ПРИМЕНЕНИЕ

[Как это будет использоваться]

## ✅ СЛЕДУЮЩИЕ ШАГИ

[План внедрения]

---

**Статус:** ✅ Завершено
```

## 📋 ЧЕК-ЛИСТ ДОБАВЛЕНИЯ В БАЗУ

### **Перед добавлением:**

```markdown
[ ] 1. Материал проанализирован (CODE пройден)
[ ] 2. Оценка ценности выставлена (1-5)
[ ] 3. Категория определена (PARA)
[ ] 4. Имя файла соответствует правилам (kebab-case)
[ ] 5. Front matter заполнен
[ ] 6. Теги добавлены
[ ] 7. Связи с другими файлами указаны
[ ] 8. Запись в журнал операций создана
```

---

### **После добавления:**

```markdown
[ ] 1. Файл закоммичен (git add + commit)
[ ] 2. Запись в OPERATION_LOG.md добавлена
[ ] 3. Навигатор обновлён (если нужно)
[ ] 4. Связи проверены (check-links.ps1)
[ ] 5. Дубликаты проверены (check-duplicates.ps1)
```

## 📊 ПРИМЕРЫ

### **Пример 1: Анализ книги**

**Книга:** "Clean Code" by Robert C. Martin

```markdown
---
title: Clean Code — ключевые идеи
source: Книга, Robert C. Martin
captured: 2026-03-03
distilled: 2026-03-03
value: 5
category: resource
tags: [код, стиль, clean-code, best-practices]
---

# 📚 CLEAN CODE — КЛЮЧЕВЫЕ ИДЕИ

## 🔑 КЛЮЧЕВЫЕ ИДЕИ

1. **Имена переменных** должны быть понятными
2. **Функции** должны быть маленькими (≤20 строк)
3. **Комментарии** — зло, код должен быть самодокументирован

## 💎 САМАЯ СУТЬ

Пишите код так, чтобы его не нужно было комментировать.

## 🎯 ПРИМЕНЕНИЕ В ПРОЕКТЕ

- Создать правило: `01_RULES/code-style.md`
- Обновить скрипты: все новые скрипты ≤20 строк
- Добавить проверку: `check-code-style.ps1`

---

**Статус:** ✅ Завершено
```
