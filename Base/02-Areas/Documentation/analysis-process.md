## 🎯 ЦЕЛЬ

Превратить сырой материал (статьи, код, ошибки) в структурированные знания, готовые к использованию в проектах.

**Три вопроса анализа:**

1. **Стоит ли сохранять?** (релевантность, качество, уникальность)
2. **Как структурировать?** (категория, формат, связи)
3. **Как описать?** (метаданные, резюме, теги)

## 📊 УРОВНИ ПРОВЕРКИ НА ДУБЛИКАТЫ

### Уровень 1: Точные дубликаты (автоматически)

**Инструмент:** `Get-FileHash` (SHA-256)

```powershell
# Вычислить хэш нового файла
$hash = Get-FileHash -Path "new_file.md" -Algorithm SHA256

# Сравнить с индексом
# Если хэш найден → точный дубликат
```

**Действие:** ❌ Отклонить (если не новая версия)

---

### Уровень 2: Похожие дубликаты (полуавтоматически)

**Инструмент:** `grep_search`

```powershell
# Поиск по заголовку
grep_search -pattern "UI Toolkit" -glob "*.md"

# Поиск по ключевым словам
grep_search -pattern "button click" -path "KNOWLEDGE_BASE/03_PATTERNS"
```

**Действие:** ⚠️ Проверить вручную (объединить или связать)

---

### Уровень 3: Семантические дубликаты (вручную)

**Инструмент:** Qwen Code (ИИ-аналитик)

**Вопросы:**
- Описывает ли ту же концепцию?
- Это развитие идеи или пересказ?
- Есть ли уникальная информация?

**Действие:** 🔄 Объединить / связать / отклонить

## 💎 ОЦЕНКА ЦЕННОСТИ

### Критерии (1-5 баллов)

| Критерий | 5 | 3 | 1 |
|----------|---|---|---|
| **Уникальность** | Нет аналогов в Базе | Есть похожие | Полный дубликат |
| **Применимость** | Будет использоваться часто | Иногда полезно | Малоприменимо |
| **Актуальность** | Соответствует текущим задачам | Может пригодиться | Устарело |
| **Качество** | Полно, точно, проверено | Есть пробелы | Неполно/неточно |

### Порог включения

- **≥4.0** → ✅ Добавить в KNOWLEDGE_BASE/
- **3.0-3.9** → ⚠️ Добавить с доработкой
- **<3.0** → ❌ Отложить/отклонить

```powershell
# scripts\weekly-knowledge-audit.ps1

# 1. Проверка дубликатов
.\check-duplicates.ps1

# 2. Проверка _drafts/ (>7 дней)
Get-ChildItem "_drafts" -Directory | Where-Object {
    $_.CreationTime -lt (Get-Date).AddDays(-7)
} | ForEach-Object {
    Write-Warning "Старый черновик: $($_.Name)"
}

# 3. Отчёт
Write-Host "Аудит завершён. Отчёт: reports\WEEKLY_AUDIT.md"
```

**Основная идея:** Безопасное удаление с бэкапом и проверкой

**Ключевые концепции:**
- Бэкап перед удалением
- Проверка целостности
- Логирование операции

**Пример кода:**
```powershell
Copy-Item $Path $BackupPath -Recurse
if (Test-Integrity) { Remove-Item $Path }
```

**Предупреждения:**
- Не удалять без бэкапа
- Проверять путь перед удалением
