## 🎯 НАЗНАЧЕНИЕ

Этот файл описывает **механизм разрешения конфликтов** между правилами Базы Знаний.

**Используйте для:**
- Обнаружения противоречий
- Разрешения конфликтов
- Приоритизации правил

---

## 📊 ТИПЫ КОНФЛИКТОВ

### Тип 1: Прямое противоречие

**Определение:** Одно правило утверждает P, другое утверждает ¬P.

**Пример:**
```
Правило А: "Всегда спрашивать выбор при переименовании"
Правило Б: "Действовать автономно при переименовании"
```

**Разрешение:**
- ✅ Правило А имеет приоритет (Красный уровень > Зелёный)

---

### Тип 2: Конфликт приоритетов

**Определение:** Два правила применяют к одной ситуации с разными действиями.

**Пример:**
```
Ситуация: Переименование файла

Правило 1 (Красный уровень): "Спросить выбор"
Правило 2 (Зелёный уровень): "Действовать автономно"
```

**Разрешение:**
- ✅ Красный уровень > Зелёный уровень

---

### Тип 3: Конфликт источников

**Определение:** Правила из разных источников противоречат друг другу.

**Пример:**
```
Источник А (QWEN.md): "Всегда спрашивать"
Источник Б (Старое правило): "Действовать самостоятельно"
```

**Разрешение:**
- ✅ QWEN.md имеет более высокий авторитет

---

### Тип 4: Временной конфликт

**Определение:** Старое и новое правило противоречат.

**Пример:**
```
Старое (2026-02-28): "Коммитить каждый час"
Новое (2026-03-02): "Коммитить крупные этапы"
```

**Разрешение:**
- ✅ Новое правило > Старое правило

---

## 🏗️ МЕХАНИЗМ РАЗРЕШЕНИЯ

### Шаг 1: Обнаружение конфликта

**Алгоритм:**
```
1. Извлечь все правила
2. Сравнить условия применения
3. Если условия пересекаются → проверить действия
4. Если действия противоречат → КОНФЛИКТ
```

**Автоматизация:**
```powershell
.\scripts\check-rule-conflicts.ps1
```

---

### Шаг 2: Классификация конфликта

**Вопросы:**
1. Это прямое противоречие (P vs ¬P)?
2. Это конфликт приоритетов?
3. Это конфликт источников?
4. Это временной конфликт?

**Ответ:** Определить тип конфликта.

---

### Шаг 3: Применение правил разрешения

**Иерархия приоритетов:**

```
1. Уровень решения (Красный > Жёлтый > Зелёный)
2. Авторитет источника (QWEN.md > другие)
3. Специфичность (Конкретное > Общее)
4. Время (Новое > Старое)
5. Приоритет файла (Глобальные > Стандарты > ...)
```

---

### Шаг 4: Разрешение конфликта

**Действия:**
1. Применить правило с высшим приоритетом
2. Пометить конфликтующее правило как "отключено"
3. Записать в OPERATION_LOG.md
4. Обновить правило (если нужно)

## 📋 ПРАВИЛА ПРИОРИТЕТИЗАЦИИ

### Правило 1: Уровни решений

**Приоритет:**
```
🔴 Красный уровень (Всегда спрашивать) — Приоритет 1
🟡 Жёлтый уровень (По контексту) — Приоритет 2
🟢 Зелёный уровень (Автономно) — Приоритет 3
```

**Пример:**
```
Конфликт:
- Красный: "Спросить при удалении"
- Зелёный: "Удалить автономно"

Разрешение: Красный уровень побеждает
```

---

### Правило 2: Авторитет источника

**Приоритет:**
```
1. .qwen/QWEN.md (Мастер-конфиг)
2. AI_START_HERE.md (Главная инструкция)
3. RULES_AND_TASKS.md (Правила)
4. 03-Resources/Knowledge/00_CORE/ (Стандарты)
5. 03-Resources/Knowledge/05_METHODOLOGY/ (Методологии)
6. 02-Areas/Documentation/ (Руководства)
7. 03-Resources/PowerShell/ (Скрипты)
```

**Пример:**
```
Конфликт:
- QWEN.md: "Только русский язык"
- Старый файл: "English allowed"

Разрешение: QWEN.md побеждает
```

---

### Правило 3: Специфичность

**Приоритет:**
```
Конкретное правило > Общее правило
Исключение > Общее правило
```

**Пример:**
```
Общее: "Коммитить каждый час"
Конкретное: "Не коммитить косметику"

Разрешение: Конкретное правило побеждает
```

---

### Правило 4: Время

**Приоритет:**
```
Новое правило > Старое правило
```

**Пример:**
```
Старое (2026-02-20): "Удалить TEMP сразу"
Новое (2026-03-02): "Бэкап перед удалением"

Разрешение: Новое правило побеждает
```

---

### Правило 5: Класс файла

**Приоритет:**
```
Глобальные правила > Стандарты > Методологии > Руководства > Скрипты
```

**Пример:**
```
Глобальное: "Вести OPERATION_LOG.md"
Скрипт: "Не записывать в логи"

Разрешение: Глобальное правило побеждает
```

## 🔄 АЛГОРИТМ РАЗРЕШЕНИЯ КОНФЛИКТОВ

```
АЛГОРИТМ Resolve-Conflict(Правило1, Правило2):

1. ОПРЕДЕЛИТЬ тип конфликта:
   ЕСЛИ P vs ¬P → Прямое противоречие
   ЕСЛИ разные действия → Конфликт приоритетов
   ЕСЛИ разные источники → Конфликт источников
   ЕСЛИ разные даты → Временной конфликт

2. ПРИМЕНИТЬ правила приоритизации:
   а) Сравнить уровни (Красный > Жёлтый > Зелёный)
   б) Сравнить авторитет источников
   в) Сравнить специфичность
   г) Сравнить даты
   д) Сравнить классы файлов

3. ВЫБРАТЬ победителя:
   Правило с наивысшим приоритетом

4. ЗАПИСАТЬ решение:
   - В OPERATION_LOG.md
   - В CONFLICT_RESOLUTION.md
   - Обновить правила (если нужно)

5. ПРОВЕРИТЬ непротиворечивость:
   Запустить check-rule-conflicts.ps1
```

## 🎯 ПРЕДОТВРАЩЕНИЕ КОНФЛИКТОВ

### При создании правила:

1. **Проверить существующие:**
   ```powershell
   grep_search -pattern "похожее правило" -glob "*.md"
   ```

2. **Запустить проверку:**
   ```powershell
   .\scripts\check-rule-conflicts.ps1
   ```

3. **Определить приоритет:**
   ```
   Какой уровень? (Красный/Жёлтый/Зелёный)
   Какой источник? (QWEN/Другой)
   Какое время? (Новое/Старое)
   ```

4. **Добавить метаданные:**
   ```markdown
   **Приоритет:** 1 (Красный)
   **Источник:** QWEN.md
   **Дата:** 2026-03-02
   ```

---

### При обновлении правила:

1. **Сравнить с другими:**
   ```
   Есть ли противоречия?
   Есть ли дубликаты?
   ```

2. **Обновить метаданные:**
   ```markdown
   **Последнее обновление:** 2026-03-02
   **Изменения:** [Описание]
   ```

3. **Записать в лог:**
   ```markdown
   ## 2026-03-02 Обновление правила
   
   **Правило:** [Название]
   **Изменения:** [Что изменено]
   **Причина:** [Почему]
   ```
