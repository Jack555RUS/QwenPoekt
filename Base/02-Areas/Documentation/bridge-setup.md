## 📋 ПРОБЛЕМА

**Текущая ситуация:**
- ✅ ИИ имеет доступ к `D:\QwenPoekt\Base`
- ❌ ИИ **НЕ** имеет доступа к:
  - `D:\QwenPoekt\Projects` (Unity проект)
  - `D:\QwenPoekt\_BACKUP` (резервные копии)
  - `D:\QwenPoekt\_TEST_ENV` (тестовая среда)
  - `D:\QwenPoekt` (корень)

**Причина:** Ограничения безопасности Qwen Code workspace.

## 🔧 РЕШЕНИЕ 1: СИМВОЛИЧЕСКИЕ ССЫЛКИ (✅ Рекомендуется)

### Что это:

Создание символических ссылок внутри `Base/` на внешние папки.

### Как создать:

#### Шаг 1: Открыть от имени администратора

```
1. Найти файл: D:\QwenPoekt\Base\create-bridge.bat
2. ПКМ → "Запуск от имени администратора"
```

#### Шаг 2: Дождаться создания ссылок

Скрипт создаст:
- `Base\_EXTERNAL` → `D:\QwenPoekt\` (корень)
- `Base\_PROJECTS` → `D:\QwenPoekt\Projects`
- `Base\_BACKUP_LINK` → `D:\QwenPoekt\_BACKUP`
- `Base\_TEST_ENV_LINK` → `D:\QwenPoekt\_TEST_ENV`

#### Шаг 3: Проверить

```
D:\QwenPoekt\Base\
├── _EXTERNAL/          ← Доступ ко всему QwenPoekt
├── _PROJECTS/          ← Доступ к Unity проектам
├── _BACKUP_LINK/       ← Доступ к бэкапам
└── _TEST_ENV_LINK/     ← Доступ к тестовой среде
```

## 🔧 РЕШЕНИЕ 2: MULTI-ROOT WORKSPACE (Альтернатива)

### Что это:

Добавление всех папок в одну VS Code workspace.

### Как настроить:

#### Шаг 1: Открыть workspace файл

```
Файл: D:\QwenPoekt\QwenPoekt.code-workspace
```

#### Шаг 2: Проверить содержимое

```json
{
    "folders": [
        {
            "path": "D:\\QwenPoekt"
        },
        {
            "path": "D:\\QwenPoekt\\Base"
        },
        {
            "path": "D:\\QwenPoekt\\Projects"
        },
        {
            "path": "D:\\QwenPoekt\\_BACKUP"
        },
        {
            "path": "D:\\QwenPoekt\\_TEST_ENV"
        }
    ]
}
```

#### Шаг 3: Открыть в VS Code

```
File → Open Workspace from File → QwenPoekt.code-workspace
```

---

## 🔧 РЕШЕНИЕ 3: JUNCTION (Без прав администратора)

### Что это:

Junction — аналог символических ссылок для локальных путей.

### Как создать:

```cmd
# Открыть cmd (не требуется администратор)
cmd

# Создать junction
mklink /J "D:\QwenPoekt\Base\_PROJECTS" "D:\QwenPoekt\Projects"
mklink /J "D:\QwenPoekt\Base\_BACKUP_LINK" "D:\QwenPoekt\_BACKUP"
mklink /J "D:\QwenPoekt\Base\_TEST_ENV_LINK" "D:\QwenPoekt\_TEST_ENV"
```

## ❓ ЧАСТЫЕ ВОПРОСЫ

### Q: Почему не работает без администратора?

**A:** Символические ссылки в Windows требуют прав администратора из соображений безопасности.

### Q: Можно ли удалить ссылки потом?

**A:** Да, просто удалите папки `_EXTERNAL`, `_PROJECTS`, `_BACKUP_LINK`, `_TEST_ENV_LINK` через проводник.

### Q: Безопасно ли это?

**A:** Да, ссылки не копируют файлы, а только создают "ярлыки" на существующие папки.
