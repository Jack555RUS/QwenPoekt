## 📋 ЧТО ПРОИСХОДИТ ПРИ ЗАПУСКЕ

### Для пользователя:

```
1. Открываете VS Code
2. Открываете папку: D:\QwenPoekt\Base
3. Открываете терминал (Ctrl+`)
4. Пишете в чат Qwen Code
```

### Для ИИ (Qwen Code):

```
1. ИИ читает системный промпт (.qwen/QWEN.md)
2. Проверяет .qwen/CURRENT_CONTEXT.md (если есть)
3. Проверяет .resume_marker.json (маркер сессии)
4. Загружает модули ядра (.qwen/rules/*.md)
5. Готов к работе!
```

## 🔄 ВОССТАНОВЛЕНИЕ СЕССИИ

### Автоматическое восстановление:

**При старте:**
```powershell
.\scripts\start-session.ps1 -Resume
```

**Или команда в чате:**
```
/resume
```

**Процесс:**
1. Проверка `.qwen/session-rules.json`
2. Проверка `.resume_marker.json`
3. Проверка `sessions/` (последняя сессия)
4. Проверка Task Scheduler (автосохранение)
5. Вывод контекста

### Ручное восстановление:

**Прочитать файлы:**
```powershell
Get-Content .resume_marker.json | ConvertFrom-Json
Get-Content reports/SESSION_HANDOVER.md  # если есть
```

## ⚠️ КРИТИЧНЫЕ ПРОВЕРКИ

### Перед началом работы:

```markdown
[✅] 1. VS Code открыт в папке D:\QwenPoekt\Base
[✅] 2. Терминал PowerShell активен
[✅] 3. Qwen Code расширение установлено
[✅] 4. .qwen/ папка существует
[✅] 5. .resume_marker.json читается
```

### Если что-то не так:

**Проблема:** Qwen Code не отвечает

**Решение:**
```
1. Проверить расширение (Ctrl+Shift+X → Qwen Code)
2. Перезагрузить VS Code (Ctrl+Shift+P → Reload Window)
3. Проверить логи (View → Output → Qwen Code)
```

**Проблема:** .qwen/ папка отсутствует

**Решение:**
```powershell
# Восстановить из Git
git checkout HEAD -- .qwen/

# Или создать заново
New-Item -ItemType Directory -Path ".qwen" -Force
```

**Проблема:** .resume_marker.json не читается

**Решение:**
```powershell
# Проверить файл
Test-Path .resume_marker.json

# Восстановить из последней сессии
Get-ChildItem sessions\ -Directory | Sort-Object CreationTime -Descending | Select-Object -First 1
```
