## 📋 3-УРОВНЕВЫЙ ПРОЦЕСС ВНЕДРЕНИЯ

### Уровень 1: Черновик (_drafts/)

**Когда:** Создание новых файлов (правила, документация, скрипты)

**Процесс:**
```
1. Создать файл в _drafts/
2. Пометить как "WIP" (Work In Progress)
3. Заполнить мета-поля (версия, дата, статус)
4. Сообщить Пользователю о готовности к тестированию
```

**Пример:**
```
_drafts/RULES_ONTOLOGY.md
  Статус: WIP
  Версия: 0.1 (черновик)
```

---

### Уровень 2: Тестирование (_TEST_ENV)

**Когда:** Файл готов к проверке

**Процесс:**
```
1. Копировать из _drafts/ в _TEST_ENV/Base/
2. Запустить test-all-changes.ps1
3. Если <100% → Исправить → Повторить
4. Если 100% → Перейти к Уровню 3
```

**Проверки:**
- ✅ Структура файлов (заголовки, мета-поля)
- ✅ Ссылки (нет битых)
- ✅ Интеграция с другими файлами
- ✅ Комплексные тесты (25/25)

---

### Уровень 3: Внедрение (Base + коммит)

**Когда:** 100% тестов пройдено

**Процесс:**
```
1. Переместить из _drafts/ в Base/
2. Обновить статус: WIP → ✅ Активно
3. Закоммитить
4. Очистить _drafts/
5. Сохранить _TEST_ENV (на случай отката)
```

**Формат коммита:**
```
Add: [Название файла] — [Описание]

Тестирование:
- _TEST_ENV: 100% (25/25 тестов)
- Ссылки: 0 битых
- Структура: ✅
```

## ✅ TDD — ПРАВИЛО ЗАВЕРШЕНИЯ ЗАДАЧИ

**Перед тем как отметить задачу выполненной:**

1. ✅ **Код компилируется** без ошибок
2. ✅ **Тесты написаны** и проходят (NUnit)
3. ✅ **EXE собирается** (если применимо)
4. ✅ **Кнопки работают** (если UI)

**Команда:** `/verify-complete` — проверить все 4 пункта

**Приоритет:** НИКОГДА не завершай задачу без проверки!

## 🔄 ЦИКЛ ПРОВЕРКИ КОДА

```
1. Написать (Clean Code, SOLID, C# 9.0+)
   → Анализаторы (StyleCop + SonarLint)

2. Протестировать (NUnit тесты)
   → Убедиться, что логика работает

3. Финальная проверка (StyleCop + SonarLint)
   → Пре-коммит анализ

4. ⚠️ ЦИКЛ ПРОВЕРКИ:
   Вопрос: Были исправления после пунктов 2 или 3?
   ├─ ДА → вернуться к пункту 2 (прогнать тесты заново)
   └─ НЕТ → перейти к пункту 5

5. Задокументировать (XML docs для public API)

6. Закоммитить (git add → commit → push)
```

**Правило:** Любое изменение после тестов → повторное тестирование!

## 🏗️ UNITY BUILD CHECKLIST

**Перед сборкой EXE:**

1. ✅ **Проверить:** Unity установлен? (`.\check-environment.ps1`)
2. ✅ **Проверить:** Лицензия активна?
3. ✅ **Запустить:** `.\auto-build-exe.ps1`
4. ✅ **Проверить:** EXE существует в Build/
5. ✅ **Запустить EXE** (если возможно)

**Команда:** `/unity-build-check` — проверить все пункты

## 🤖 MULTI-AGENT MODE

**Для сложных задач использовать 4 агентов параллельно:**

| Агент | Роль |
|-------|------|
| **Agent 1 (Code)** | Пишет код |
| **Agent 2 (Tests)** | Пишет тесты |
| **Agent 3 (Docs)** | Обновляет документацию |
| **Agent 4 (Build)** | Проверяет сборку |

**Команда:** `/multi-agent [задача]` — запустить 4 агентов

**Правило:** Задача не завершена, пока Agent 4 не подтвердит сборку
