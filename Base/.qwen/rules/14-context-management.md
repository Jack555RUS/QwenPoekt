# 14 — УПРАВЛЕНИЕ КОНТЕКСТОМ (CONTEXT MANAGEMENT)

**Версия:** 1.0
**Дата:** 4 марта 2026 г.
**Статус:** ✅ Активно
**Модуль:** `.qwen/rules/14-context-management.md`

---

## 🎯 НАЗНАЧЕНИЕ

Это правило определяет механизмы управления контекстом диалога:
1. ✅ **Query Refinement** (переформулирование запросов)
2. ✅ **Summarization** (суммаризация истории)
3. ✅ **Switch Notification** (уведомления о смене темы)
4. ✅ **Token Management** (экономия токенов)

**Главный принцип:** 20 минут на управление контекстом > 2 часа исправлений

---

## 📋 7 ШАГОВ УПРАВЛЕНИЯ КОНТЕКСТОМ

### ШАГ 1: МОНИТОРИНГ КОНТЕКСТА

**Правило:** Проверять заполненность контекста каждые 10 сообщений

**Пороги:**
| Порог | Действие |
|-------|----------|
| **>25%** (32,000 токенов) | ⚠️ Предупреждение |
| **>50%** (64,000 токенов) | 🔴 Суммаризация |
| **>75%** (96,000 токенов) | 🔴🔴 Срочная суммаризация |
| **>90%** (115,000 токенов) | 🔴🔴🔴 Блокировка (риск потери) |

**Команда:**
```powershell
.\scripts\summarize-context.ps1 -Threshold 0.5 -Verbose
```

---

### ШАГ 2: QUERY REFINEMENT (переформулирование)

**Правило:** Переформулировать запросы с местоимениями

**Паттерн:**
```
Запрос: "А сколько там студентов?"
Контекст: "Томский государственный университет"
История: ["Расскажи о ТГУ", "А сколько там студентов?"]
↓
Уточнённый запрос: "Какое количество студентов обучается 
                    в Томском государственном университете?"
```

**Алгоритм:**
1. Найти местоимения ("это", "там", "он", "она", "тот")
2. Извлечь сущности из контекста
3. Заменить местоимения на сущности
4. Добавить контекст в запрос

**Формула:**
```
Уточнённый запрос = Запрос + Контекст + История (последние 3 сообщения)
```

---

### ШАГ 3: SUMMARIZATION (суммаризация)

**Правило:** Суммаризировать при >50% заполненности

**Пороги:**
- **>50% контекста** → Суммаризировать
- **>1000 токенов на сообщение** → Удалить/сжать
- **>10 сообщений без темы** → Перегруппировать

**Формат суммаризации:**
```markdown
## Краткое содержание
[3-5 предложений]

## Ключевые факты
- Факт 1
- Факт 2
- Факт 3

## Решения
- Решение 1
- Решение 2
```

**Команда:**
```powershell
.\scripts\summarize-context.ps1 -Threshold 0.5 -SummaryLength 5
```

---

### ШАГ 4: SWITCH NOTIFICATION (уведомление о смене темы)

**Правило:** Уведомлять о переключении темы

**Формат:**
```markdown
💡 Переключаемся на тему: [Название]

Предыдущая тема: [Сохранено в контексте]
Новая тема: [Описание]
Связанные файлы: [Список]
```

**Пример:**
```markdown
💡 Переключаемся на тему: INDEX.md

Предыдущая тема: Управление контекстом (сохранено в summary.md)
Новая тема: Создание единого индекса структуры базы
Связанные файлы: INDEX.md, BASE_STRUCTURE_MAP.md
```

---

### ШАГ 5: TOKEN MANAGEMENT (экономия токенов)

**Правило:** Удалять старые сообщения, сохранять ключевые факты

**Алгоритм:**
```
1. Подсчитать токены
2. Если >50% → Суммаризировать
3. Старые сообщения (>1000 токенов назад) → Удалить
4. Ключевые факты → Сохранить в key-facts.json
5. Суммаризацию → Сохранить в summary.md
```

**Экономия:**
| До | После | Экономия |
|-----|-------|----------|
| 100,000 токенов | 40,000 токенов | 60% |
| 80,000 токенов | 35,000 токенов | 56% |
| 120,000 токенов | 50,000 токенов | 58% |

---

### ШАГ 6: CONTEX SUMMARY (резюме контекста)

**Правило:** Обновлять резюме после каждой суммаризации

**Файл:** `.resume_marker.json`

**Поля:**
```json
{
  "ContextSummary": "Обсуждали INDEX.md. Решили: 8 разделов...",
  "ContextSummaryDate": "2026-03-04 20:00",
  "ContextTokens": 40000,
  "ContextThreshold": 0.5
}
```

**Команда:**
```powershell
# Автоматическое обновление через summarize-context.ps1
```

---

### ШАГ 7: ПРЕДУПРЕЖДЕНИЯ (alerts)

**Правило:** Предупреждать о критичных состояниях

| Состояние | Предупреждение | Действие |
|-----------|----------------|----------|
| **>75% токенов** | 🔴 "Контекст заполнен на 75%" | Суммаризировать сейчас |
| **>90% токенов** | 🔴🔴 "Критично: 90% токенов!" | Немедленная суммаризация |
| **Местоимения без контекста** | ⚠️ "Неясный запрос" | Query Refinement |
| **>10 сообщений без темы** | ⚠️ "Тема не определена" | Перегруппировать |

## ⚠️ ПОДВОДНЫЕ КАМНИ

### Проблема 1: Потеря важных деталей

**Описание:** При суммаризации теряются числа, имена

**Решение:**
- Использовать `Extract-KeyFacts`
- Сохранять в `key-facts.json`

---

### Проблема 2: Избыточная суммаризация

**Описание:** Суммаризация при <50% (неоправданные затраты)

**Решение:**
- Проверять порог перед суммаризацией
- Пропускать при <50%

---

### Проблема 3: Непонимание местоимений

**Описание:** "А сколько там?" — непонятно, где "там"

**Решение:**
- Query Refinement перед обработкой
- Передача контекста + истории
