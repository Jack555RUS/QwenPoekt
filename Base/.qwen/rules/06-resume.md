# 🔄 КОМАНДА /RESUME — Восстановление Сессии

**Версия:** 1.0
**Дата:** 2 марта 2026 г.
**Статус:** ✅ Активно

---

## 📋 НАЗНАЧЕНИЕ

Команда `/resume` предназначена для:
1. ✅ **Автоматического восстановления** контекста сессии
2. ✅ **Проверки предыдущей сессии** (маркер, файлы, Git)
3. ✅ **Вывода сводки** о последней задаче
4. ✅ **Предложения продолжения** с последнего шага

---

## 🚀 ИСПОЛЬЗОВАНИЕ

### Базовое использование:

```
/resume
```

**Что происходит:**
1. Проверка `.resume_marker.json`
2. Чтение `SESSION_HANDOVER.md`
3. Проверка Git статуса
4. Вывод контекста для ИИ
5. Предложение продолжения

---

### Расширенные команды:

| Команда | Описание |
|---------|----------|
| **`/resume`** | Полное восстановление сессии |
| **`/resume --verbose`** | С подробной диагностикой |
| **`/resume --force`** | Принудительное (игнорировать маркер) |
| **`/status`** | Показать статус сессии без восстановления |
| **`/continue`** | Продолжить с последней задачи (без диагностики) |

---

## 🧠 ЛОГИКА ИИ ПРИ ВОССТАНОВЛЕНИИ

### При получении команды `/resume`:

**1. Проверка маркера:**
```
✅ Маркер найден → Чтение данных
❌ Маркер не найден → Проверка SESSION_HANDOVER.md
❌ Оба не найдены → "Нет предыдущей сессии"
```

**2. Извлечение контекста:**
```
- Последняя задача
- Текущий шаг
- Следующее действие
- Активные задачи
- Проблемы/ошибки
```

**3. Форматирование ответа:**
```
╔══════════════════════════════════════════════════════════╗
║              КОНТЕКСТ ВОССТАНОВЛЕН! ✅                   ║
╚══════════════════════════════════════════════════════════╝

📊 Информация о сессии:
  Дата: 2026-03-02 21:45
  Причина: Перезагрузка (RAM диск)
  Задача: Установка RAM диска
  След. шаг: 4/6 — Настройка переменных среды

📋 Активные задачи:
  1. RAM диск (шаг 4/6) 🔴
  2. Диалог выхода (префаб) 🟡

💡 Рекомендация:
  Продолжить с шага 4 (настройка TEMP на R:)?
```

## 🎯 ТРИГГЕРЫ АВТОСОХРАНЕНИЯ

### Слова-триггеры (автоматическое сохранение):

**Выход:**
- "выход", "exit", "quit", "bye"
- "до свидания", "goodbye", "see you"
- "конец сессии", "завершаю"

**Перезагрузка:**
- "перезагрузка", "reboot", "restart"
- "выключение", "shutdown", "power off"
- "перезагрузиться", "rebooting"

**Прерывание:**
- "стоп", "stop", "break", "pause"
- "прервать", "interrupt", "hold on"

### При обнаружении триггера:

**Диалог:**

## 🔧 ТЕХНИЧЕСКАЯ РЕАЛИЗАЦИЯ

### Скрипты:

| Скрипт | Назначение |
|--------|------------|
| `auto-save-session.ps1` | Автосохранение при триггерах |
| `start-session.ps1` | Проверка при старте |
| `end-session.ps1` | Ручное завершение (legacy) |

### Файлы:

| Файл | Назначение |
|------|------------|
| `.resume_marker.json` | Маркер восстановления (JSON) |
| `reports/SESSION_HANDOVER.md` | Контекст сессии (Markdown) |
| `ТЕКУЩАЯ_ЗАДАЧА.md` | Активные задачи |

### Формат маркера (.resume_marker.json):

```json
{
  "Date": "2026-03-02 21:45:00",
  "Reason": "Reboot for RAM disk",
  "CurrentTask": "Установка RAM диска",
  "NextStep": "4/6 — Настройка переменных среды",
  "SessionFile": "reports/SESSION_HANDOVER.md",
  "GitCommit": "abc123"
}
```

## 🛠️ ОТЛАДКА

### Диагностика:

```powershell
# Проверка маркера
Test-Path .resume_marker.json

# Просмотр маркера
Get-Content .resume_marker.json | ConvertFrom-Json

# Проверка SESSION_HANDOVER.md
Test-Path reports/SESSION_HANDOVER.md

# Запуск диагностики
.\scripts\start-session.ps1 -Verbose

# Ручное автосохранение
.\scripts\auto-save-session.ps1 -Reason "Manual" -CurrentTask "Тест" -NextStep "Шаг 1"
```

### Типичные проблемы:

| Проблема | Решение |
|----------|---------|
| **Маркер не найден** | Проверить путь (корень Base/) |
| **SESSION_HANDOVER.md пуст** | Запустить auto-save-session.ps1 |
| **Git ошибки** | Проверить `git status` |
| **/resume не работает** | Проверить интеграцию в AGENTS.md |
