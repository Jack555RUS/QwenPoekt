## 🎯 НАЗНАЧЕНИЕ

Это правило предотвращает действия с недостаточными знаниями.

**Главный принцип:** 20 минут на изучение > 2 часа исправлений

## 📋 ЧЕК-ЛИСТ ПРОВЕРКИ

### 1. Оценка текущих знаний

**Вопросы:**
- [ ] Что я уже знаю об этой теме?
- [ ] Какой у меня опыт (0/1/5/10 лет)?
- [ ] Могу ли объяснить это другому?
- [ ] Все ли аспекты задачи охвачены?
- [ ] Понимаю ли я ограничения?

**Решение:**
- Если опыт <3 лет → Изучить минимум 3 источника
- Если не можешь объяснить → Изучить ещё
- Если <5 аспектов охвачено → Углубиться

**Не начинать действие если:**

- 🔴 Изучено <3 источников

## 💡 ПРИМЕРЫ

### Пример 1: Настройка Vector DB

**Задача:** "Настроить Vector DB для RAG"

**Анализ:**
```
1. Текущие знания:
   - Известно: Что такое RAG, базовое понимание векторов
   - Неизвестно: Какие DB существуют, совместимость с Python 3.14
   - Опыт: 0 лет (первый раз)
   - Самооценка: 3/10

2. Источники:
   - docs.python.org — совместимость библиотек
   - GitHub chroma-core/chroma — 15000 звёзд
   - Stack Overflow — вопросы по установке
   - Reddit r/Python — обсуждения

3. Варианты:
   - A: Chroma DB (просто, но проблемы с Python 3.14)
   - B: Qdrant (сложнее, но быстрее)
   - C: Упрощённая версия без DB (pickle)

4. Риски:
   - Chroma DB несовместима → Потеря времени
   - Qdrant требует Docker → Сложность

5. Выбор:
   - Вариант C (pickle) — быстро, работает, без зависимостей

6. Решение:
   - Достаточно знаний: ✅ Да
   - Можно начинать: ✅ Да
```

**Результат:** vectorize-kb-simple.py создан за 1 час
