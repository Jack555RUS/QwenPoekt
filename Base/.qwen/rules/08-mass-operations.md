## 🟣 ФИОЛЕТОВЫЙ УРОВЕНЬ — ВСЕГДА СПРАШИВАТЬ

**Когда:**
- Перемещение **>100 файлов**
- Переименование **>50 файлов**
- Реорганизация **структуры проекта**
- Удаление **>10 файлов/папок**
- Изменение **конфигурации ИИ**

**Формат вопроса:**
```
⚠️ ТРЕБУЕТСЯ ПОДТВЕРЖДЕНИЕ

Это массовая операция (>100 файлов).

План:
1. Запустить pre-move-check.ps1
2. Получить подтверждение пользователя
3. Выполнить перемещение
4. Обновить ссылки (update-links-after-move.ps1)
5. Закоммитить изменения

Продолжить? (Y/N)
```

**Действие:** Ждать подтверждения!

## 📋 ПРОЦЕСС МАССОВОГО ПЕРЕМЕЩЕНИЯ

### **Шаг 1: Проверка (pre-move-check.ps1)**

```powershell
.\scripts\pre-move-check.ps1 `
  -SourcePath "scripts/" `
  -DestinationPath "03-Resources/PowerShell/" `
  -Filter "*.ps1" `
  -Verbose
```

**Проверяет:**
- ✅ Количество файлов (<100 или >100)
- ✅ Существование путей
- ✅ Ссылки на файлы (битые ссылки)
- ✅ Dry-run перемещение
- ✅ Отчёт о влиянии

---

### **Шаг 2: Подтверждение пользователя**

**ИИ задаёт вопрос:**
```
⚠️ МАССОВОЕ ПЕРЕМЕЩЕНИЕ

📊 Файлов: 76
💾 Размер: 500 KB
🔗 Ссылок: 20 (будут обновлены)

Продолжить? (Y/N)
```

**Действие:** Ждать ответа!

---

### **Шаг 3: Выполнение перемещения**

```powershell
# После подтверждения
Move-Item -Path "scripts/*.ps1" -Destination "03-Resources/PowerShell/" -Force
```

**Важно:**
- ✅ Перемещать с флагом `-Force`
- ✅ Сохранять структуру папок
- ✅ Проверять успех после каждого файла

---

### **Шаг 4: Обновление ссылок**

```powershell
.\scripts\update-links-after-move.ps1 `
  -OldPath "scripts/" `
  -NewPath "03-Resources/PowerShell/"
```

**Обновляет:**
- ✅ Ссылки в .md файлах
- ✅ Ссылки в .ps1 файлах
- ✅ Ссылки в .json файлах

---

### **Шаг 5: Коммит**

```powershell
git add .
git commit -m "Move: 76 файлов из scripts/ в 03-Resources/PowerShell/"
```

**Формат коммита:**
```
Move: [количество] файлов из [старый путь] в [новый путь]

Проверка:
- pre-move-check.ps1: ✅
- update-links-after-move.ps1: ✅
- Ссылок обновлено: [количество]
```

## 🛑 СТОП ТРИГГЕРЫ (НЕМЕДЛЕННО ПРЕКРАТИТЬ)

**Триггеры:**

### **Процесс внедрения в Ядро:**

```
1. 7 вопросов (выше) → Заполнить
2. Тестирование в _TEST_ENV/ → Пройдено
3. Бэкап (git commit + auto-archive) → Создан
4. Чек-лист безопасности → Проверен
5. Внедрение в Base/ → Выполнено
6. Проверка (check-kernel-integrity.ps1) → Пройдено
7. Коммит → Создан
```
