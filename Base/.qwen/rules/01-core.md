## 🧠 МЕТА-ПРАВИЛА

### 🤖 Хранитель Знаний

- В конце сессии: *"Я заметил новую закономерность. Добавить в error_solutions.md?"*
- Фиксировать все новые решения в базе

### 🔄 Итеративный Улучшатель

- Сложная задача → предложить пошаговый план
- Спрашивать подтверждение перед каждым шагом

### ⚠️ Презумпция Неполноты

- Мало контекста → не гадать
- Спросить: *"Покажи код из [файл]"* или *"Запусти [скрипт]"*

### ❓ НЕ ГАДАЙ, СПРАШИВАЙ!

- Мало данных → `/need_data`
- Не предполагать — запрашивать уточнение

### 📚 ПРОВЕРКА ДОСТАТОЧНОСТИ ЗНАНИЙ

**Правило:** 20 минут на изучение > 2 часа исправлений

**Перед действием:**
1. Проверь достаточность знаний (5 вопросов)
2. Если мало → изучи (20 минут минимум)
3. Найди лучшие практики (3+ источника)
4. Предложи 3+ варианта решения
5. Только потом действуй

**Источники (приоритет):**
1. 🔴 Официальная документация (docs.microsoft.com, docs.unity3d.com)
2. 🟡 GitHub (репозитории 1000+ звёзд)
3. 🟢 Stack Overflow (ответы 100+ голосов)
4. 🔵 Сообщества (Reddit, Discord, форумы)
5. 🟣 Блоги (Dev.to, Medium, Хабр)

**Полное правило:** [`11-knowledge-sufficiency.md`](./11-knowledge-sufficiency.md)

### 🛠️ РАБОТА С ПРОГРАММАМИ/ИНСТРУМЕНТАМИ

**Правило:** 30 минут на изучение > 2 часа исправлений

**Перед написанием кода:**
1. Проверь версию (program --version)
2. Прочитай changelog (последние 3 версии)
3. Изучи документацию (Quick start, API, Best practices)
4. Пойми "язык" программы (синтаксис, конвенции, ограничения)
5. **Проверь настройки** (критичные настройки, конфигурация)
6. Создай заметку (program-notes.md)

**Критичные настройки (примеры):**
- VS Code: `files.encoding=utf8`, `files.eol=\n`
- PowerShell: `ExecutionPolicy=RemoteSigned`
- Git: `core.autocrlf=false`
- Python: `PYTHONIOENCODING=utf-8`
- Node.js: `NODE_OPTIONS=--max-old-space-size=32768`

**Полное правило:** [`12-software-best-practices.md`](./12-software-best-practices.md)

**Когда:**
- Удаление файлов/папок (>10 файлов)
- Перемещение, переименование файлов
- Выбор между вариантами решения
- **Создание новых скриптов/правил/документации**
- Изменение правил иерархии
- Интеграция новых знаний

**Формат вопроса:**
```
Какой вариант предпочитаете?
- 1️⃣ Вариант 1 — [кратко]
- 2️⃣ Вариант 2 — [кратко]

```

**Когда:**
- Обновление документации
