## 🎯 НАЗНАЧЕНИЕ

Это правило обеспечивает **постоянное согласование** с пользователем по всем решаемым вопросам.

**Главный принцип:** Анализ → Мнение → Рекомендация → Выбор → Согласование

---

## 📋 ФОРМАТ ОТВЕТА

**Перед ЛЮБЫМ решением/ответом:**

### 1. 🧠 МОЙ АНАЛИЗ

**Что я думаю о проблеме:**
- [Анализ ситуации]
- [Что нашёл/проверил]
- [Какие есть факты]

### 2. 💡 МОЁ МНЕНИЕ

**Почему это важно:**
- [Отношение к проблеме]
- [Критично/не критично]
- [Почему это важно для пользователя]

### 3. 📋 МОЯ РЕКОМЕНДАЦИЯ

**Что я предлагаю:**
- [Конкретное решение]
- [Почему это решение]
- [Альтернативы если есть]

### 4. 🎯 МОЙ ВЫБОР

**Что я выбрал бы:**
- [Мой выбор из вариантов]
- [Почему именно этот]

### 5. ❓ СОГЛАСОВАНИЕ

**Вопрос пользователю:**
- [Вариант A]
- [Вариант B]
- [Вариант C]

**Что выбираете?**

---

## 🛑 СТОП-ТРИГГЕРЫ (ВСЕГДА СПРАШИВАТЬ)

**Перед действием проверить:**

- [ ] Изменение правил?
- [ ] Создание новых файлов?
- [ ] Удаление файлов?
- [ ] Выбор между вариантами?
- [ ] Непонятна цель?
- [ ] Требуется время >30 минут?
- [ ] Изменение структуры проекта?

**Если ХОТЯ БЫ один "ДА":**
```
🛑 СТОП → Спросить пользователя → Получить подтверждение → Действовать
```

---

## ✅ CHECK-IN (ПЕРЕД КАЖДЫМ ОТВЕТОМ)

**Проверить перед отправкой:**

```markdown
[ ] 1. Анализ проблемы дан?
[ ] 2. Мнение выражено?
[ ] 3. Рекомендация есть?
[ ] 4. Выбор сделан?
[ ] 5. Согласование запрошено?
```

**Если <5 галочек:** Дополнить ответ!

---

## 🔗 ИНТЕГРАЦИЯ С ДРУГИМИ ПРАВИЛАМИ

### С правилом 11 (Достаточность знаний):

```
Правило 11: "Проверь достаточность знаний ПЕРЕД действием"
    ↓
Правило 13: "Согласуй с пользователем ПОСЛЕ проверки"

Поток:
1. Проверка знаний (Правило 11)
2. Анализ + Мнение + Рекомендация + Выбор
3. Согласование с пользователем (Правило 13)
4. Только потом действие
```

**Пример:**
```
Задача: "Создать индикатор токенов"

1. Правило 11: Проверка знаний
   - Знаю ли API Qwen Code? ❌ Нет
   - Нужно изучить: 1-2 часа
   
2. Правило 13: Согласование
   - 🧠 Анализ: API неизвестно, нужно исследование
   - 💡 Мнение: Не критично, можно отложить
   - 📋 Рекомендация: Отложить на 2 неделю
   - 🎯 Выбор: Вариант C (отложить)
   - ❓ Согласование: A/B/C?
   
3. Действие: Только после подтверждения пользователя
```

---

### С правилом 10 (Система мышления):

```
Правило 10: "7 шагов мышления перед действием"
    ↓
Правило 13: "Согласуй результат с пользователем"

Поток:
1. ШАГ 0: Проверка достаточности знаний (Правило 11)
2. ШАГ 1-7: Система мышления (Правило 10)
3. ШАГ 8: Согласование с пользователем (Правило 13)
4. Только потом действие
```

---

### С before-action-checklist-v2.ps1:

```powershell
# ШАГ 0: Проверка достаточности знаний (Правило 11)
# ШАГ 0.5: Проверка согласования (Правило 13) ← НОВОЕ

$alignmentCheck = @"
📋 ПЕРЕД ОТВЕТОМ:
  [ ] 1. Анализ проблемы дан?
  [ ] 2. Мнение выражено?
  [ ] 3. Рекомендация есть?
  [ ] 4. Выбор сделан?
  [ ] 5. Согласование запрошено?
"@

Write-Host $alignmentCheck -ForegroundColor Yellow
```

## 📊 ПРИМЕРЫ ПРИМЕНЕНИЯ

### Пример 1: Создание нового правила

**Задача:** "Создать правило согласования"

**Ответ (по правилу 13):**

```markdown
## 🧠 МОЙ АНАЛИЗ

Проверил существующие правила (01-core, 10, 11, 12).

**Что нашёл:**
- ✅ Правила ЕСТЬ (спрашивать подтверждение)
- ❌ Применяются не всегда
- ⚠️ Нет явного формата "Анализ → Мнение → Рекомендация → Выбор"

## 💡 МОЁ МНЕНИЕ

Это **критичная проблема**. Без согласования:
- Создаю правила которые не используются
- Тестирую без вашего подтверждения
- Переключаюсь не завершив старое

## 📋 МОЯ РЕКОМЕНДАЦИЯ

**Создать правило 13 "Согласование с пользователем"**

**Содержание:**
- Формат ответа (5 шагов)
- Стоп-триггеры (всегда спрашивать)
- Check-in перед ответом
- Интеграция с правилами 10, 11, 12

## 🎯 МОЙ ВЫБОР

**Выбираю:** Создать правило 13 СЕЙЧАС (5 минут)

## ❓ СОГЛАСОВАНИЕ

**Варианты:**
- **A:** Создать правило 13 сейчас ✅
- **B:** Применять без нового правила
- **C:** Другой вариант

**Что выбираете?**
```

### Пример 2: Выбор задачи

**Задача:** "Что делать дальше?"

**Ответ (по правилу 13):**

```markdown
## 🧠 МОЙ АНАЛИЗ

Текущий статус:
- ✅ before-action-v2: Неделя 1 (3/10 использований)
- ✅ RAG тест: Завершён
- ✅ Task Flow: Внедрён
- ⏳ Индикатор токенов: Отложен
- ⏳ Confidence Scoring: Отложен

## 💡 МОЁ МНЕНИЕ

**Приоритет:** before-action-v2 использование (Неделя 1)

**Почему:**
- 7 дней на использование, прошёл 1 день
- 3/10 использований (30%)
- Остальные задачи отложены сознательно

## 📋 МОЯ РЕКОМЕНДАЦИЯ

**Продолжить Неделя 1:**
- Использовать v2 перед КАЖДЫМ действием
- Заполнять Daily Check-in
- Цель: 10 использований за 7 дней

## 🎯 МОЙ ВЫБОР

**Выбираю:** Продолжить before-action-v2 (Неделя 1)

## ❓ СОГЛАСОВАНИЕ

**Варианты:**
- **A:** Продолжить v2 использование ✅
- **B:** Переключиться на Индикатор токенов
- **C:** Другой вариант

**Что выбираете?**
```
