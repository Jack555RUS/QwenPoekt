## ❌ ПРОБЛЕМЫ (из опыта)

### 1. Действия без анализа

**Пример:**
```
Задача: "Исправить битую ссылку"
Действие: Удалить ссылку
Последствие: Потерян функционал, ссылка вела на переименованный файл
```

**Почему:** Не проанализировал **ЗАЧЕМ** ссылка была создана

---

### 2. Непроверенный синтаксис

**Пример:**
```
Задача: "Написать скрипт"
Действие: Написать код с ошибками
Последствие: 5+ итераций исправлений
```

**Почему:** Не проверил документацию перед написанием

---

### 3. Сдача при ошибках

**Пример:**
```
Задача: "Скачать файл"
Действие: Одна попытка → ошибка → отказ
Правильно: 3+ попытки, альтернативные источники
```

---

### 4. Работа над ошибками "для галочки"

**Пример:**
```
Ошибка → Исправил → Забыл
Правильно: Ошибка → Исправил → Протестировал → Лог → Правило → Автоматизация
```

## ✅ РЕШЕНИЕ: 7-ШАГОВАЯ СИСТЕМА

### ШАГ 1: СТОП (обязательно перед действием)

**Правило:** НИКАКОГО кода/изменений без паузы!

**Чек-лист:**
```markdown
[ ] 1. Остановился на 30 секунд
[ ] 2. Глубокий вдох-выдох
[ ] 3. Задал вопрос: "Зачем я это делаю?"
```

---

### ШАГ 2: АНАЛИЗ КОНТЕКСТА

**Вопросы:**
```
1. ЗАЧЕМ это нужно? (цель)
2. ЧТО уже есть? (существующие решения)
3. КТО это использует? (зависимости)
4. ГДЕ это применяется? (контекст)
5. КОГДА это нужно? (приоритет)
```

**Действия:**
```powershell
# Поиск существующих решений
grep_search "pattern" -Path .
glob "**/*.ps1"

# Проверка зависимостей
check-links.ps1
check-duplicates.ps1
```

---

### ШАГ 3: ИЗУЧЕНИЕ ДОКУМЕНТАЦИИ

**Правило:** 20 минут на изучение > 2 часа исправлений

**Чек-лист:**
```markdown
[ ] 1. Проверил версию ПО
[ ] 2. Прочитал официальную документацию
[ ] 3. Изучил примеры использования
[ ] 4. Проверил changelog (последние исправления)
[ ] 5. Сохранил заметки (03-Resources/AI/program-notes.md)
```

**Пример заметки:**
```markdown
# Python 3.14 + Chroma DB

**Проблема:** Несовместимы (pydantic v1)
**Решение:** Использовать vectorize-kb-simple.py (без Chroma)
**Дата:** 2026-03-04
**Источник:** 03-Resources/AI/python-compatibility-notes.md
```

---

### ШАГ 4: ПРОВЕРКА ГИПОТЕЗ

**Метод:** 3 попытки минимум

**Алгоритм:**
```
Попытка 1: Основное решение
  ↓ (ошибка)
Попытка 2: Альтернатива
  ↓ (ошибка)
Попытка 3: Упрощённая версия
  ↓ (ошибка)
Анализ: Почему не работает? Поиск аналогов
```

**Пример:**
```
Задача: "Скачать файл"

Попытка 1: Прямая ссылка → timeout (30 сек)
Попытка 2: Зеркало → файл не найден
Попытка 3: Альтернативный источник → успех

ИЛИ

Анализ: Может файл не нужен? Есть ли аналог?
```

---

### ШАГ 5: ПРЕДВАРИТЕЛЬНЫЙ ТЕСТ

**Правило:** Тест ДО коммита/внедрения

**Чек-лист:**
```markdown
[ ] 1. Запустил в тестовой среде (_TEST_ENV/)
[ ] 2. Проверил логи
[ ] 3. Проверил зависимости
[ ] 4. Проверил обратную совместимость
[ ] 5. Записал результат теста
```

---

### ШАГ 6: АНАЛИЗ ПОСЛЕДСТВИЙ

**Вопросы:**
```
1. Что сломается если это удалить?
2. Что зависит от этого?
3. Где это используется?
4. Как откатить если что-то пойдёт не так?
```

**Действия:**
```powershell
# Поиск зависимостей
grep_search "ссылка" -Path .
build-link-graph.ps1

# Проверка перед удалением
before-action-validator.ps1 -Action delete -Target "file.md"
```

### ШАГ 7: ТЕСТИРОВАНИЕ + ЛОГИРОВАНИЕ

```markdown
[ ] 1. Запустил тесты
```

## 🛡️ АВТОМАТИЗАЦИЯ

### Скрипт 1: before-action-checklist.ps1

```powershell
# Проверка перед действием
param($action, $target)

Write-Host "╔══════════════════════════════════════════════════════════╗"
Write-Host "║         ПРОВЕРКА ПЕРЕД ДЕЙСТВИЕМ                         ║"
Write-Host "╚══════════════════════════════════════════════════════════╝"

# 1. Анализ зависимостей
Write-Host "`n[1/5] Поиск зависимостей..."
$dependencies = grep_search $target -Path . -Include "*.md"
Write-Host "  Найдено ссылок: $($dependencies.Count)"

# 2. Проверка версии
Write-Host "`n[2/5] Проверка версии..."
python --version
node --version

# 3. Тест в _TEST_ENV
Write-Host "`n[3/5] Тест в _TEST_ENV..."
# Копирование и тест

# 4. Логирование
Write-Host "`n[4/5] Логирование..."
$logPath = "logs\action-$(Get-Date -Format 'yyyyMMdd-HHmmss').log"
Start-Transcript $logPath

# 5. Подтверждение
Write-Host "`n[5/5] Требуется подтверждение"
$confirm = Read-Host "Продолжить? (Y/N)"
if ($confirm -ne 'Y') { exit 0 }

Write-Host "`n✅ Проверка завершена"
```
