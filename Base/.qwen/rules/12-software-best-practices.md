**Главный принцип:** 30 минут на изучение > 2 часа исправлений

---

## 📋 ЧЕК-ЛИСТ (ПЕРЕД НАЧАЛОМ РАБОТЫ)

**6 шагов (40 минут):**

1. Проверить версию (5 минут)
2. Прочитать changelog (10 минут)
3. Изучить документацию (15 минут)
4. Понять "язык" программы (10 минут)
5. **Проверить настройки** (10 минут) ← НОВОЕ
6. Создать заметки (10 минут)

**Вопросы:**
- [ ] Какая версия установлена?
- [ ] Какая версия последняя?
- [ ] Есть ли обновления?
- [ ] Есть ли breaking changes?

**Команды:**
```powershell
# Проверка версии
program --version
python --version
node --version
code --version

# Проверка последней версии (GitHub)
curl https://api.github.com/repos/owner/repo/releases/latest
```

**Решение:**
- Если есть обновление → Обновить (если не breaking)
- Если breaking changes → Прочитать migration guide

---

### 2. Changelog (последние 3 версии) (10 минут)

**Что искать:**
- Breaking changes
- Новые фичи
- Исправления ошибок
- Устаревшие функции (deprecated)

**Где искать:**
- `CHANGELOG.md` в репозитории
- GitHub Releases
- Официальный сайт (Release Notes)

**Решение:**
- Если есть breaking changes → Изучить migration guide
- Если есть новые фичи → Рассмотреть для использования

---

### 3. Официальная документация (15 минут)

**Прочитать обязательно:**
- [ ] Quick start guide
- [ ] API reference / синтаксис
- [ ] Best practices
- [ ] Common pitfalls / troubleshooting

**Где искать:**
- `docs.[program].com`
- `[program].org/docs`
- GitHub Wiki

**Решение:**
- Сделать пометки в program-notes.md

---

### 4. Изучение "языка" программы (10 минут)

**Что изучать:**

#### Синтаксис:
- Как объявляются переменные/функции
- Как обрабатываются ошибки
- Как импортируются модули

#### Конвенции:
- Именование (camelCase, PascalCase, snake_case)
- Структура файлов
- Стиль кода

#### Ограничения:
- Что не поддерживает
- Известные баги
- Ограничения производительности

**Решение:**
- Записать в program-notes.md

---

### 5. Проверка настроек программы (10 минут)

**Вопросы:**
- [ ] Конфигурационный файл найден?
- [ ] Настройки актуальны?
- [ ] Есть ли специфичные настройки для нашей системы?

**Критичные настройки (примеры):**

| Программа | Настройка | Значение | Почему |
|-----------|-----------|----------|--------|
| **VS Code** | `files.encoding` | `utf8` | Без BOM для русских символов |
| **VS Code** | `files.eol` | `\n` | LF для Git совместимости |
| **PowerShell** | `ExecutionPolicy` | `RemoteSigned` | Скрипты работают |
| **Git** | `core.autocrlf` | `false` | Не менять концы строк |
| **Python** | `PYTHONIOENCODING` | `utf-8` | Unicode в консоли |
| **Node.js** | `NODE_OPTIONS` | `--max-old-space-size=32768` | Память для MCP |

**Конфигурационные файлы:**
- `VS Code:` `.vscode/settings.json`
- `PowerShell:` `$PROFILE`
- `Git:` `.gitconfig`
- `Python:` `pyproject.toml`, `.python-version`
- `Node.js:** `package.json`, `.nvmrc`

**Решение:**
- Если настройки устарели → Обновить
- Если нет настроек → Создать (по шаблону)
- Записать в program-notes.md

## 🔍 ИСТОЧНИКИ (ПРИОРИТЕТ)

### 🔴 Уровень 1: Официальные источники

| Источник | Что искать | Когда |
|----------|------------|-------|
| **Официальный сайт** | Документация, скачать | Всегда |
| **GitHub Releases** | Версии, changelog | Перед началом |
| **Официальные docs** | API, синтаксис, примеры | Перед написанием кода |
| **Migration guides** | Breaking changes | При обновлении |

**Примеры:**
- `code.visualstudio.com/docs` — VS Code
- `docs.python.org` — Python
- `powershell.microsoft.com/docs` — PowerShell
- `unity.com/manual` — Unity
- `nodejs.org/docs` — Node.js

---

| Источник | Что искать | Критерии |
|----------|------------|----------|
| **GitHub Issues** | Проблемы, решения | Закрытые issues |
| **GitHub Discussions** | Обсуждения, Q&A | Ответы от maintainers |
| **Stack Overflow** | Вопросы по программе | Тег программы |
| **Reddit** | Обсуждения, новости | r/программа |
| **Discord/Slack** | Чат сообщества | Официальный сервер |

**Как искать:**
```

### Пример 1: VS Code MCP SDK

**Задача:** "Написать MCP сервер для VS Code"

**Анализ:**
```
1. Версия:
   - VS Code: 1.87.0 ✅
   - Node.js: 20.11.0 ✅
   - MCP SDK: 1.x (breaking changes от 0.x) ⚠️

2. Changelog:
   - MCP SDK 1.x: Новый API (McpServer вместо Server)
   - Требуется @modelcontextprotocol/sdk 1.x

3. Документация:
   - modelcontextprotocol.io/docs ✅
   - GitHub: modelcontextprotocol/specification ✅

4. "Язык" программы:
   - TypeScript/JavaScript
   - Импорт: import { McpServer } from '@modelcontextprotocol/sdk'

   - Использовать Zod для валидации
   - Типизировать все параметры

6. Примеры:
   - GitHub: modelcontextprotocol/servers
   - Официальные примеры

7. Риски:
   - Breaking changes → Использовать SDK 1.x
   - Несовместимость → Проверить package.json

   - Достаточно информации: ✅ Да
   - Заметка создана: 03-Resources/AI/vscode-mcp-notes.md

**Анализ:**
```

   - PowerShell: 7.4.1 ✅
   - Тернарный оператор: Только в 7.0+ ⚠️

   - PowerShell 7.0: Добавлен тернарный оператор
   - PowerShell 7.4: Улучшена производительность

   - docs.microsoft.com/powershell ✅
   - PowerShell.org ✅

4. "Язык" программы:
   - Синтаксис: PowerShell (не Bash!)
   - Конвенции: PascalCase для команд
   - Ограничения: Нет некоторых Bash фич

5. Лучшие практики:
   - Использовать Write-Host для вывода
   - Использовать param() для параметров
   - Обрабатывать ошибки через try/catch

6. Примеры:
   - GitHub: PowerShell/PowerShell
   - Официальные скрипты

7. Риски:
   - Тернарный оператор не работает в старых версиях
   - Решение: Использовать if/else

8. Решение:
   - Достаточно информации: ✅ Да
   - Заметка создана: 03-Resources/AI/powershell-notes.md
```
