## 📖 МЕТОД: Feynman Technique (Объясни как новичку)

### **Шаг 1: Выбери концепцию**
**Тема:** Обработка информации для ИИ-ассистента

### **Шаг 2: Объясни как 5-летнему**
**Простыми словами:**
```
Информация — это как игрушки в комнате.

Без системы:
- Игрушки везде (пол, кровать, шкаф)
- Когда нужно найти — долго ищем
- Некоторые теряются навсегда

С системой (PARA):
- Игрушки по коробкам (куклы, машинки, конструктор)
- Каждая коробка подписана
- Знаем, где что лежит

Zettelkasten — это как дружить с игрушками:
- Каждая игрушка знает другие игрушки
- Машинка знает, где гараж
- Кукла знает, где домик
- Все связаны между собой

Evergreen — это расти вместе с игрушками:
- Сегодня играем в машинки
- Завтра добавляем гараж
- Послезавтра добавляем заправку
- Игра становится больше и лучше
```

### **Шаг 3: Выяви пробелы**
**Что непонятно:**
1. ❓ Как именно связывать заметки в Zettelkasten?
2. ❓ Когда переводить из Inbox в Permanent?
3. ❓ Как часто делать обзор (Review)?

### **Шаг 4: Упрости и создай аналогии**
**Аналогии для ИИ-ассистента:**

| Концепция | Аналогия | Реализация |
|-----------|----------|------------|
| **PARA** | Папки на рабочем столе | `01-Projects/`, `02-Areas/` |
| **Zettelkasten** | Связи между идеями | `[[ссылка]]` в Markdown |
| **Evergreen** | Растущий сад | Заметки эволюционируют |
| **Inbox** | Входящие почты | `_inbox/` (временная папка) |

### **3. Read (Чтение с планом)**

**План чтения:**
```
1. PARA (5 мин) → Основа структуры
2. Zettelkasten (10 мин) → Связи между идеями
3. Evergreen (10 мин) → Эволюция знаний
4. Гибрид (15 мин) → Применение для ИИ
```

**Результат чтения:**

#### **PARA — Основа (Действие)**
```
Projects → Текущие задачи (с дедлайном)
Areas → Постоянные обязанности
Resources → Справочные материалы
Archives → Завершённое
```

**Для ИИ:**
- `01-Projects/AutoSave-Feature/` — текущая задача
- `02-Areas/Code-Quality/` — стандарты
- `03-Resources/PowerShell/` — справочник
- `04-Archives/2026-02/` — архив

---

#### **Zettelkasten — Связи (Атомарность)**
```
Одна заметка = одна идея
Связи через [[ссылки]]
Сеть знаний растёт
```

**Для ИИ:**
```markdown
# PowerShell Error Handling Pattern

## Идея
Try-Catch-Finally для обработки ошибок

## Связи
- [[PowerShell Best Practices]]
- [[Error Handling Patterns]]
- [[Automation Scripts]]

## Пример
try { ... } catch { ... } finally { ... }
```

---

#### **Evergreen — Эволюция (Рост)**
```
Заметки созревают со временем
Заголовки-утверждения
Готовый материал для документации
```

**Для ИИ:**
```markdown
# Автосохранение каждые 2 минуты экономит токены

**Создано:** 2026-03-02
**Обновлено:** 2026-03-03

## Эволюция
- v1: Автосохранение (2 мин)
- v2: Добавлен тихий режим (VBScript)
- v3: Интеграция с Task Scheduler

## Связи
- [[Token Optimization]]
- [[Session Management]]
```

---

### **4. Recite (Пересказ)**

**Что запомнил (простыми словами):**

1. **PARA** — это 4 коробки для всего:
   - Делаем сейчас (Projects)
   - Поддерживаем всегда (Areas)
   - Может пригодиться (Resources)
   - Закончили (Archives)

2. **Zettelkasten** — это сеть идей:
   - Одна заметка = одна мысль
   - Связываем между собой
   - Сеть растёт умнее

3. **Evergreen** — это живой сад:
   - Сажаем семя (идея)
   - Поливаем (обновляем)
   - Растёт и цветёт (документация)

4. **Гибрид для ИИ:**
   - PARA сверху (структура)
   - Zettelkasten внутри (связи)
   - Evergreen для роста (эволюция)

---
