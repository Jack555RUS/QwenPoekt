## 📋 3-УРОВНЕВЫЙ ПРОЦЕСС ВНЕДРЕНИЯ

### Уровень 1: Черновик (_drafts/)

**Когда:** Создание новых файлов (правила, документация, скрипты)

**Процесс:**
```
1. Создать файл в _drafts/
2. Пометить как "WIP" (Work In Progress)
3. Заполнить мета-поля (версия, дата, статус)
4. Сообщить Пользователю о готовности к тестированию
```

**Пример:**
```
_drafts/RULES_ONTOLOGY.md
  Статус: WIP
  Версия: 0.1 (черновик)
```

---

### Уровень 2: Тестирование (_TEST_ENV)

**Когда:** Файл готов к проверке

**Процесс:**
```
1. Копировать из _drafts/ в _TEST_ENV/Base/
2. Запустить test-all-changes.ps1
3. Если <100% → Исправить → Повторить
4. Если 100% → Перейти к Уровню 3
```

**Проверки:**
- ✅ Структура файлов (заголовки, мета-поля)
- ✅ Ссылки (нет битых)
- ✅ Интеграция с другими файлами
- ✅ Комплексные тесты (25/25)

---

### Уровень 3: Внедрение (Base + коммит)

**Когда:** 100% тестов пройдено

**Процесс:**
```
1. Переместить из _drafts/ в Base/
2. Обновить статус: WIP → ✅ Активно
3. Закоммитить
4. Очистить _drafts/
5. Очистить _TEST_ENV (сохранить структуру)
```

**Формат коммита:**
```
Add: [Название файла] — [Описание]

Тестирование:
- _TEST_ENV: 100% (25/25 тестов)
- Ссылки: 0 битых
- Структура: ✅
```

### 🔹 6. GIT / ВЕРСИОНИРОВАНИЕ:

**Коммит (когда ДА):**

**Бэкап (по запросу):**
- `/backup` — коммит + push на GitHub
- `/compare` — сравнить с GitHub
- `/status` — показать статус

**Напоминание:**
- ⚠️ **Перед завершением сессии** → спросить: "Закоммитить важное?"
- ⚠️ **Несохранённые изменения** → предупредить

**Автоматизация:**
- `auto-commit-daily.ps1` — ежедневный коммит (18:00)
- Push — по необходимости (крупные изменения)

**Правило PUSH:**
- ✅ **Ежедневно:** Коммит локально (auto-commit-daily.ps1 в 18:00)
- ✅ **Крупные изменения:** PUSH сразу (завершил задачу, исправил ошибку)
- ✅ **По запросу:** `/backup` — коммит + push
- ❌ **Не нужно:** PUSH каждый день (ждём крупного изменения)

**Проверка:**
```powershell
git status
```
- `ahead by X commits` → Можно push (если крупное изменение)
- `up to date` → PUSH не нужен

### Stack Overflow: База знаний "вопрос-ответ"

**Правило:** Обращай внимание не только на принятый ответ, но и на комментарии!

**Как использовать:**

1. **Читай вопрос полностью:**
   - Пойми контекст проблемы
   - Проверь, та ли это проблема, что у тебя

2. **Анализируй ответы:**
   - ✅ **Принятый ответ** (зелёная галочка) — работает у автора
   - ✅ **Ответы с высоким рейтингом** (много голосов) — работает у многих
   - ⚠️ **Комментарии под ответами** — там часто обсуждаются подводные камни!

3. **Что искать в комментариях:**
   - "This works, but..." — работает, но есть нюансы
   - "This is deprecated..." — устарело, не используй
   - "Better approach..." — есть решение лучше
   - "Security issue..." — проблема безопасности

4. **Проверяй дату:**
   - ✅ Ответы за последний год — актуальны
   - ⚠️ Ответы 2-3 года — проверяй актуальность
   - ❌ Ответы 5+ лет — скорее всего устарели
