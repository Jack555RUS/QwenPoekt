### Вариант 1: MCP инструмент (1-2 часа)

**Описание:** Создать MCP сервер для мониторинга токенов

**Требования:**
- Доступ к API Qwen Code (неизвестно есть ли)
- Интеграция в before-action-checklist-v2.ps1

**Код:**
```javascript
// mcp-token-monitor.js
server.registerTool('get_token_usage', {
  description: 'Получить использование токенов сессии',
  inputSchema: {}
}, async () => {
  const usage = await getTokenUsage();
  return {
    used: usage.used,
    total: usage.total,
    remaining: usage.total - usage.used,
    percent: (usage.used / usage.total * 100).toFixed(1)
  };
});
```

### Вариант 4: Ручной подсчёт (1 час) ⭐ РЕКОМЕНДУЕТСЯ

**Описание:** Скрипт считает токены в диалоге

**Преимущества:**
- ✅ Просто (30 минут)
- ✅ Работает сразу
- ✅ Не требует API

**Недостатки:**
- ⚠️ Приблизительный подсчёт (1 токен ≈ 4 символа)
- ⚠️ Не учитывает системный промпт

**Код:**
```powershell
# count-tokens.ps1
param($filePath)

$content = Get-Content $filePath -Raw
$tokens = [math]::Round($content.Length / 4)
$total = 128000 # Qwen-2.5-Coder
$remaining = $total - $tokens
$percent = [math]::Round($tokens / $total * 100, 1)

Write-Host "╔══════════════════════════════════════════════════════════╗"
Write-Host "║ ИСПОЛЬЗОВАНИЕ ТОКЕНОВ                                    ║"
Write-Host "╚══════════════════════════════════════════════════════════╝"
Write-Host "Использовано: $tokens токенов ($percent%)"
Write-Host "Осталось: $remaining токенов" -ForegroundColor $(
    if ($remaining -lt 10000) { "Red" }
    elseif ($remaining -lt 50000) { "Yellow" }
    else { "Green" }
)
```

**Статус:** ✅ Готов к реализации
