## 📋 2. КЛАССИФИКАЦИЯ ОШИБОК (адаптировано)

### 2.1: Ошибки в содержимом базы знаний

| Тип | Пример | Скрипт проверки |
|-----|--------|-----------------|
| **Фактические** | Неверные данные | `check-links.ps1` |
| **Логические** | Противоречия | `find-duplicates.ps1` |
| **Неточности** | Расплывчатые формулировки | `before-action-checklist-v2.ps1` |
| **Орфография** | Опечатки | Code Spell Checker |

---

### 2.2: Ошибки в структуре и связях

| Тип | Пример | Скрипт проверки |
|-----|--------|-----------------|
| **Битые ссылки** | Ссылка на несуществующий файл | `check-links.ps1` |
| **Неправильная категоризация** | Файл не в той папке | `find-duplicates.ps1` |
| **Отсутствие связей** | Файл не связан с другими | `build-link-graph.ps1` |

---

### 2.3: Ошибки в ответах ИИ

| Тип | Пример | Метрика |
|-----|--------|---------|
| **Галлюцинации** | Модель выдаёт информацию которой нет | RAGAS (faithfulness) |
| **Неполнота** | Ответ не охватывает все аспекты | RAGAS (answer relevancy) |
| **Неверный источник** | Поиск выдал нерелевантные документы | RAGAS (context precision) |

---

### 2.4: Ошибки в процессах

| Тип | Пример | Предотвращение |
|-----|--------|----------------|
| **Пропуск обновления** | Источник изменился, база не обновлена | Мониторинг источников |
| **Нарушение процедур** | Документ добавлен без проверки | `before-action-checklist-v2.ps1` |
| **Недостаточное тестирование** | Изменение без тестов | `test-assertions.ps1` |

## 🔍 3. ПРОЦЕСС СБОРА И АНАЛИЗА ОШИБОК

### 3.1: Источники информации об ошибках

| Источник | Что фиксируем | Где |
|----------|---------------|-----|
| **Логи сессий** | Ошибки в ответах ИИ | `sessions/` |

| **No-match запросы** | Вопросы без ответа | `sessions/` |

---

### 3.2: Регистрация ошибок (трекинг)

**Формат записи в ERROR_LOG.md:**

```markdown
### [ERR-XXX] Краткое название ошибки

**Дата:** YYYY-MM-DD  
**Категория:** [Содержимое/Структура/Процессы/ИИ]  
**Серьёзность:** 🔴 Критичная / 🟡 Средняя / 🟢 Косметика  
**Статус:** ✅ Исправлено / ⏳ В работе / 🔴 Не исправлено

#### Описание
[Что случилось?]

#### Корневая причина (5 Почему)
1. Почему? → [...]
2. Почему? → [...]
3. Почему? → [...]
4. Почему? → [...]
5. Почему? → [Корневая причина]

#### Влияние
[Что пострадало? Сколько времени потеряно?]

#### Решение
[Как исправили?]

#### Извлечённый урок
[Что нужно запомнить?]

#### Предотвращение (Prevention)
[Какое правило/скрипт/проверка создана?]

#### Связанные файлы
- [`путь/к/файлу.md`](путь/к/файлу.md)
```

---

### 3.3: Анализ первопричин (Root Cause Analysis, RCA)

**Метод "5 Почему":**

```
Ошибка: ИИ сообщил неверную ставку НДС

1. Почему? → В базе был устаревший документ
2. Почему? → Источник (сайт ФНС) изменился, но мы не отследили
3. Почему? → Нет автоматического мониторинга изменений
4. Почему? → Не настроена подписка на RSS
5. Почему? → Не было в плане работ

Корневая причина: Отсутствие автоматизированного мониторинга критических источников
```

**Предотвращение:**
- ✅ Настроить мониторинг сайтов
- ✅ Добавить тест в Golden Set
- ✅ Обновить процедуру добавления документов

## 🛡️ 4. МЕХАНИЗМЫ ПРЕДОТВРАЩЕНИЯ ПОВТОРЕНИЯ

### 4.1: Корректирующие действия (исправление конкретной ошибки)

| Действие | Когда | Пример |
|----------|-------|--------|
| Исправить неверный факт | Сразу | Обновить документ |
| Обновить ссылки | Сразу | `check-links.ps1` |
| Переобучить модель | По необходимости | RAGAS метрики |
| Удалить дубликат | Сразу | `find-duplicates.ps1` |

---

### 4.2: Предупреждающие действия (изменение процессов)

| Действие | Когда | Пример |
|----------|-------|--------|
| Автоматизация проверок | После ошибки | `test-assertions.ps1` |
| Улучшение мониторинга | После упущения | Мониторинг источников |
| Обновление процедур | После нарушения | `before-action-checklist-v2.ps1` |
| Валидация данных | После ошибки | Golden Set тесты |

---

### 4.3: Обновление политик и стандартов

**Когда:**
- Ошибка выявила пробел в стандартах
- Отсутствие требования

**Пример:**
```
Было: Документ добавляется без проверки
Стало: Документ проходит before-action-checklist-v2.ps1
```

---

### 4.4: Обучение команды

**Форматы:**
- Разбор ошибки на собрании
- Памятка в базе знаний
- Тренинг по инструментам

**Где:**
- `ERROR_LEARNING_GUIDE.md` (это руководство)
- Внутренняя вики

---

### 4.5: Обратная связь в цикл разработки

**Ошибочные случаи → Новые тесты:**

```
1. Ошибка в ответе ИИ
   ↓
2. Добавить вопрос в Golden Set
   ↓
3. Запустить регрессионный тест
   ↓
4. Убедиться что ошибка не повторится
```

- Категория: Содержимое
- Серьёзность: 🟡 Средняя

**Анализ первопричин:**
```
1. Почему? → В базе устаревший документ
2. Почему? → Источник изменился, не отследили
3. Почему? → Нет мониторинга
4. Почему? → Не было в плане

Корневая причина: Отсутствие процесса мониторинга источников
```

- ✅ Настроить мониторинг источников
- ✅ Добавить тест в Golden Set
