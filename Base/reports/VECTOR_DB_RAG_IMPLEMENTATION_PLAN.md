## 📖 ТЕОРИЯ (ИЗ ДОКУМЕНТАЦИИ)

### Что такое RAG?

**RAG (Retrieval-Augmented Generation)** — поиск + генерация:

```
Вопрос → Поиск в БД → Контекст + Вопрос → LLM → Ответ
```

**Пример:**
```
"Какая погода?" → Поиск → "Лондон, 15°C" → LLM → "В Лондоне 15°C"
```

**Преимущества:**
- ✅ Актуальные данные
- ✅ Меньше галлюцинаций
- ✅ Расширение контекста

**Недостатки:**
- ❌ Требует БД
- ❌ Задержка на поиск

## 🏗️ АРХИТЕКТУРА РЕШЕНИЯ

### Вариант 1: Chroma DB (Локальная)

```
┌─────────────────────────────────────────────────────────┐
│  ДОКУМЕНТЫ (.md файлы)                                  │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  EMBEDDINGS (векторизация)                              │
│  • sentence-transformers (Python)                       │
│  • all-MiniLM-L6-v2 (384 измерения)                     │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  CHROMA DB (векторное хранилище)                        │
│  • Локальное хранение                                   │
│  • Быстрый поиск                                        │
│  • Метаданные (путь, дата, теги)                        │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  RAG PIPELINE                                           │
│  1. Вопрос → векторизация                               │
│  2. Поиск похожих (cosine similarity)                   │
│  3. Контекст + Вопрос → LLM                             │
│  4. Ответ                                               │
└─────────────────────────────────────────────────────────┘
```

**Преимущества:**
- ✅ Локальное хранение (без облака)
- ✅ Быстрый старт (простая настройка)
- ✅ Python (много библиотек)
- ✅ Бесплатно

**Недостатки:**
- ❌ Требует Python
- ❌ Дополнительный процесс
