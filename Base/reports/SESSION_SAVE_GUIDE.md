## 📊 КАК ЭТО РАБОТАЕТ

### Система автосохранения

```
┌─────────────────────────────────────────────────────────┐
│  Task Scheduler (Windows)                               │
│  Запускает каждые 2 минуты                              │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  auto-save-chat.ps1                                     │
│  Проверяет конфигурацию                                 │
│  Создаёт папку сессии                                   │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  sessions/YYYY-MM-DD_HH-mm/                             │
│  chat.md (переписка)                                    │
│  metadata.json (метаданные)                             │
└─────────────────────────────────────────────────────────┘
```

### MCP сервер (прямое сохранение)

```
┌─────────────────────────────────────────────────────────┐
│  Qwen Code (ИИ)                                         │
│  Вызывает MCP инструмент                                │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  mcp-session-saver.js                                   │
│  MCP сервер для сохранения                              │
└─────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────┐
│  sessions/YYYY-MM-DD_HH-mm/                             │
│  chat.md, metadata.json, commands.log                   │
└─────────────────────────────────────────────────────────┘
```

## 📋 КОМАНДЫ

### В чате Qwen Code:

| Команда | Действие |
|---------|----------|
| `/save-session` | **Сохранить текущую сессию** (чат + метаданные) |
| `/auto-save on` | Включить автосохранение |
| `/auto-save off` | Выключить автосохранение |
| `/auto-save status` | Показать статус автосохранения |
| `/resume` | Восстановить сессию |
| `/status` | Показать статус сессии |

### PowerShell:

```powershell
# Аудит системы
.\scripts\audit-session-save.ps1

# Очистка пустых сессий (проверка)
.\scripts\cleanup-empty-sessions.ps1 -DryRun

# Очистка пустых сессий (удаление)
.\scripts\cleanup-empty-sessions.ps1 -AutoConfirm

# Проверка Task Scheduler
Get-ScheduledTask -TaskName "QwenSessionAutoSave"
```

## 🔄 ПРОЦЕСС СОХРАНЕНИЯ

### Автоматическое сохранение (каждые 2 минуты)

1. Task Scheduler запускает `auto-save-chat.ps1`
2. Скрипт проверяет `.qwen/session-rules.json`
3. Создаётся папка `sessions/YYYY-MM-DD_HH-mm/`
4. Сохраняется `chat.md` и `metadata.json`

**Проблема:** Диалог ИИ не передаётся в скрипт → пустой шаблон

### Ручное сохранение (рекомендуется)

**Перед завершением сессии:**

```
/save-session
```

**Или через PowerShell:**

```powershell
.\scripts\save-chat-log.ps1
```

## ⚠️ ЧАСТЫЕ ПРОБЛЕМЫ

### Проблема 1: Пустые сессии

**Симптом:** Сессии содержат только шаблон (7 строк)

**Причина:** Диалог ИИ не передаётся в скрипт автосохранения

**Решение:**
1. Использовать `/save-session` вручную
2. Дождаться исправления MCP интеграции

---

### Проблема 2: MCP сервер не подключается

**Симптом:** MCP инструменты недоступны

**Причина:** VS Code не активировал MCP сервер

**Решение:**
1. Проверить `mcp.json` (конфигурация)
2. Перезапустить VS Code
3. Проверить логи MCP

---

### Проблема 3: Task Scheduler не запускает

**Симптом:** Автосохранение не работает

**Причина:** Задача не создана или отключена

**Решение:**
```powershell
# Проверить задачу
Get-ScheduledTask -TaskName "QwenSessionAutoSave"

# Создать задачу (если нет)
.\scripts\schedule-backup-tasks.ps1
```
