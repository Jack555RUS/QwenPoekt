## 🧠 КАК ЭТО РАБОТАЕТ

### Автосохранение (при выходе):

```powershell
# Автоматически при триггерах: "выход", "перезагрузка", "выключение"
.\scripts\auto-save-session.ps1 -Reason "User exit" -CurrentTask "Задача" -NextStep "Шаг"
```

**Что происходит:**
1. ✅ Сбор информации о сессии (Git, задачи, файлы)
2. ✅ Создание SESSION_HANDOVER.md v2
3. ✅ Архивация сессии в _LOCAL_ARCHIVE
4. ✅ Создание маркера .resume_marker.json
5. ✅ Коммит (опционально)

---

### Восстановление (при старте):

```powershell
# Автоматическая проверка
.\scripts\start-session.ps1 -Resume

# Или команда в чате
/resume
```

**Что происходит:**
1. ✅ Проверка маркера .resume_marker.json
2. ✅ Чтение SESSION_HANDOVER.md
3. ✅ Проверка Git статуса
4. ✅ Проверка активных задач
5. ✅ Вывод контекста для ИИ

---

### Триггеры автосохранения:

**Слова-триггеры (автоматически):**
- "выход", "exit", "quit", "bye"
- "перезагрузка", "reboot", "restart"
- "выключение", "shutdown", "power off"
- "конец", "end", "stop", "break"
- "до свидания", "goodbye"

**При обнаружении триггера:**
```
ИИ → Предложение: "Сохранить контекст сессии?"
Пользователь → "Да" → auto-save-session.ps1
```

## 🛠️ ОТЛАДКА

### Проблемы и решения:

| Проблема | Решение |
|----------|---------|
| **Маркер не найден** | Проверить `.resume_marker.json` в корне Base |
| **SESSION_HANDOVER.md пуст** | Запустить `auto-save-session.ps1` вручную |
| **Git ошибки** | Проверить `git status` |
| **Команда /resume не работает** | Проверить интеграцию в AGENTS.md |

### Диагностика:

```powershell
# Проверка маркера
Test-Path .resume_marker.json

# Проверка SESSION_HANDOVER.md
Test-Path reports/SESSION_HANDOVER.md

# Просмотр маркера
Get-Content .resume_marker.json | ConvertFrom-Json

# Запуск диагностики
.\scripts\start-session.ps1 -Verbose
```
