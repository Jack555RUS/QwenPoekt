## 🎯 НАЗНАЧЕНИЕ

Эта система управляет потоком задач от создания до завершения с проверками.

**Главный принцип:** Задача ≠ Запись. Задача = Результат + Проверка.

## 📋 СТАТУСЫ ЗАДАЧ

| Статус | Значение | Критерии |
|--------|----------|----------|
| **🗓️ Backlog** | Ожидает | Записана, не приоритетна |
| **📅 This Week** | Приоритет недели | Выбрана для текущей недели |
| **📌 Today** | Сегодня | Будет сделана сегодня |
| **⏳ Doing** | В работе | Максимум 2 задачи |
| **🔍 Review** | На проверке | Ждёт проверки DoD |
| **✅ Done** | Завершено | DoD проверено |

---

## ✅ DEFINITION OF DONE (DoD)

**Задача считается завершённой ТОЛЬКО если:**

```markdown
[ ] 1. Запись в файле создана
[ ] 2. Результат на практике получен
[ ] 3. Отчёт о выполнении создан
[ ] 4. Тесты пройдены (если применимо)
[ ] 5. Закоммичено и запушено
[ ] 6. Проверено независимым лицом (скрипт/вы)
```

**Пример:**

| Задача | 1 | 2 | 3 | 4 | 5 | 6 | Статус |
|--------|---|---|---|---|---|---|--------|
| RAG тест | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ Done |
| v2 создание | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ⏳ Review |
| Индикатор токенов | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ | 🗓️ Backlog |
