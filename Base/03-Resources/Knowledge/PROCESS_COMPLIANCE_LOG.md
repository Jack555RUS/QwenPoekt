## 🚨 ТИПЫ НАРУШЕНИЙ

### Тип 1: Пропущен черновик

**Описание:** Файл создан сразу в Base/

**Пример:**
```
Запрос: "Создай правило"
Нарушение: Создано в Base/KNOWLEDGE_BASE/
Правильно: Создать в _drafts/ → Тесты → Base/
```

**Исправление:**
```
1. Переместить в _drafts/
2. Копировать в _TEST_ENV
3. Запустить тесты
4. Если 100% → Переместить в Base/
5. Закоммитить отчёт о тестировании
```

---

### Тип 2: Пропущено тестирование

**Описание:** Файл создан в _drafts/, но не тестировался

**Пример:**
```
Запрос: "Создай 4 файла"
Нарушение: _drafts/ → Base/ (без _TEST_ENV)
Правильно: _drafts/ → _TEST_ENV → Тесты → Base/
```

**Исправление:**
```
1. Копировать в _TEST_ENV
2. Запустить test-all-changes.ps1
3. Если 100% → Закоммитить
4. Если <100% → Исправить → Повторить
```

---

### Тип 3: Неправильная категоризация

**Описание:** Документация классифицирована как "косметика" (Зелёный уровень)

**Пример:**
```
Запрос: "Создай документацию"
Нарушение: "Косметика → Зелёный уровень → Автономно"
Правильно: "Системная документация → Красный уровень → Спросить"
```

**Исправление:**
```
1. Остановиться
2. Спросить подтверждение
3. Следовать процессу
```
