## 1. 🎯 НАЗНАЧЕНИЕ

**Эпизодическая память** — система хранения целых цепочек рассуждений с полным контекстом ситуации.

**Основная идея:**
> "Запомнить не только решение, но и ПУТЬ к решению: контекст, рассуждения, ошибки, инсайты"

**Отличие от Case-Based Reasoning:**
- **CBR:** Проблема → Решение (структурированный кейс)
- **Эпизодическая память:** Контекст → Цепочка рассуждений → Результат (полная история)

## 3. 🔄 ПРОЦЕСС ЗАПОМИНАНИЯ

### 5 этапов:

```
1. ЗАХВАТ (Capture)
   └─→ Записать полную цепочку рассуждений в реальном времени
   
2. СТРУКТУРИРОВАНИЕ (Structure)
   └─→ Оформить по шаблону эпизода
   
3. ИНДЕКСАЦИЯ (Index)
   └─→ Добавить теги, категории, ключевые слова
   
4. СВЯЗЫВАНИЕ (Link)
   └─→ Связать с другими эпизодами и кейсами
   
5. КОНСОЛИДАЦИЯ (Consolidate)
   └─→ Сохранить в долговременную память
```

## 4. 🔍 ПОИСК И ИЗВЛЕЧЕНИЕ

### Критерии поиска:

| Критерий | Описание |
|----------|----------|
| **Контекст** | Похожая ситуация, проект, версия |
| **Тип задачи** | UI, логика, оптимизация, отладка |
| **Структура рассуждений** | Похожий путь решения |
| **Инсайты** | Общие закономерности |

### Функция поиска:

```python
def retrieve_episodic_memory(new_situation, threshold=0.8):
    """
    Извлечь похожие эпизоды из памяти
    
    Args:
        new_situation: Описание текущей ситуации
        threshold: Порог相似ности (0.0-1.0)
    
    Returns:
        List[episode]: Отсортированные эпизоды с цепочками рассуждений
    """
    similar_episodes = []
    
    for episode in EPISODIC_MEMORY:
        # Сравнение контекста
        context_similarity = compare_context(new_situation, episode.context)
        
        # Сравнение структуры рассуждений
        reasoning_similarity = compare_reasoning_chains(
            new_situation.reasoning, 
            episode.reasoning_chain
        )
        
        # Общая相似ность
        total_similarity = (
            0.4 * context_similarity + 
            0.6 * reasoning_similarity
        )
        
        if total_similarity >= threshold:
            similar_episodes.append((episode, total_similarity))
    
    return sorted(similar_episodes, key=lambda x: x[1], reverse=True)
```

### EPISODE-002: Исправление потери данных при очистке _TEMP

**Дата:** 2 марта 2026 г.  
**Длительность:** 45 минут  
**Сессия:** SESSION_2026-03-02_06-34

### 🎯 Исходная задача:
> "При тестировании 2 марта потеряны ~21 скрипт PowerShell и ~37 отчётов из-за досрочного закрытия терминала при очистке _TEMP. Нужно предотвратить повторение."

### 🧠 ЦЕПОЧКА РАССУЖДЕНИЙ:

#### Шаг 1: Первичный анализ
- **Что увидел:**
  - Пользователь закрыл терминал до завершения скрипта
  - _TEMP очищен до завершения копирования
  - 21 скрипт и 37 отчётов потеряны
  
- **Гипотеза 1:**
  - "Нужно добавить проверку завершения перед очисткой"
  
- **Результат:**
  - ✅ Частично подтверждено

#### Шаг 2: Углубление
- **Что нашёл:**
  - Нет явного завершения сессии
  - Пользователь не знает, когда скрипт завершён
  - Нет финального отчёта
  
- **Гипотеза 2:**
  - "Нужна команда завершения сессии с автоматической очисткой"
  
- **Результат:**
  - ✅ Подтверждено

#### Шаг 3: Решение
- **Финальное решение:**

```powershell
# end-session.ps1 — завершение сессии
# 1. Создать отчёт
# 2. Закоммить важное
# 3. Очистить _TEMP
# 4. Показать финальный статус

Write-Host "✅ Сессия завершена"
Write-Host "📦 _TEMP очищен"
Write-Host "💾 Git обновлён"
```

- **Результат:**
  - ✅ Скрипт создан
  - ✅ Команда "завершаем сессию" добавлена

### 💡 КЛЮЧЕВЫЕ ИНСАЙТЫ:

1. **"Явное завершение"** — команда должна быть однозначной
2. **"Автоматический отчёт"** — фиксировать результат сессии
3. **"Защита от досрочного закрытия"** — проверка перед очисткой

## 6. 🧩 ИНТЕГРАЦИЯ С ДРУГИМИ СИСТЕМАМИ

### Связь с Case-Based Reasoning:

```
Эпизодическая память (полная цепочка)
         ↓
    Извлечение инсайтов
         ↓
Case-Based Reasoning (структурированный кейс)
```

**Процесс:**
1. Эпизод сохраняется с полной цепочкой
2. Анализ выделяет ключевые паттерны
3. Создаётся кейс для CBR базы
4. Оба файла связаны ссылками

---

### Связь с индукцией:

```
Эпизод 1: Контекст A → Решение X
Эпизод 2: Контекст B → Решение X
Эпизод 3: Контекст C → Решение X
         ↓
    Индуктивный вывод
         ↓
Правило: "Если A/B/C → использовать X"
```
