## 💡 СОВЕТЫ ПО ИСПОЛЬЗОВАНИЮ

### 1. Не копируйте, а понимайте

**Правило:**
```
❌ Плохо: Скопировать код ИИ без понимания
✅ Хорошо: Попросить объяснить код (/explain)
```

**Команда:**
```
/explain этот код
```

---

### 2. Используйте режимы правильно

| Задача | Режим |
|--------|-------|
| Написание кода | Принятия правок (Tab) |
| Форматирование | YOLO |
| Сложная задача | Планирования |
| Вопрос | Обычный (>) |

---

### 3. Пишите понятные запросы

**Плохо:**
```
сделай функцию
```

**Хорошо:**
```
создай функцию GetPlayerHealth() которая:
- возвращает текущее здоровье игрока
- проверяет на null
- логирует предупреждение если здоровье < 20%
```

---

### 4. Используйте контекст

**Правило:**
```
✅ Выделите код перед запросом
✅ Укажите файл с которым работать
✅ Дайте контекст задачи
```

---

### 5. Проверяйте код ИИ

**Чек-лист:**
```markdown
[ ] Код компилируется
[ ] Логика правильная
[ ] Нет уязвимостей
[ ] Есть обработка ошибок
[ ] Соответствует стандартам проекта
```

---

## 🔧 КОМАНДЫ ИИ

### Популярные команды:

| Команда | Назначение |
|---------|------------|
| `/explain` | Объяснить код |
| `/fix` | Исправить ошибку |
| `/tests` | Создать тесты |
| `/doc` | Сгенерировать документацию |
| `/refactor` | Рефакторинг |
| `/optimize` | Оптимизация |

---

## ⚠️ ЧАСТЫЕ ОШИБКИ

| Ошибка | Как избежать |
|--------|--------------|
| **Слепое копирование** | Всегда просить объяснение |
| **Нет контекста** | Выделять код, давать контекст |
| **Не проверяют код** | Запускать тесты после ИИ |
| **Один режим для всего** | Использовать правильный режим |
