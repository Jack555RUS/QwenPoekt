## 🎯 НАЗНАЧЕНИЕ

Эта папка предназначена для хранения **устаревших, но исторически ценных** материалов Базы Знаний.

**Что сюда перемещать:**

- ✅ Устаревшие правила (заменены новыми версиями)
- ✅ Исторические паттерны (не актуальны, но полезны для истории)
- ✅ Старые версии скриптов (для сравнения)
- ✅ Архивные статьи (заменены новыми)

**Что НЕ перемещать:**

- ❌ Актуальные материалы
- ❌ Черновики (для них `_drafts/`)
- ❌ Временные файлы (удалить)

## 🔄 ПРОЦЕСС АРХИВИРОВАНИЯ

### Шаг 1: Определение кандидата

**Критерии:**

- Материал заменён новой версией
- Материал устарел (технология, подход)
- Материал исторически ценен (для истории проекта)

---

### Шаг 2: Перемещение

**Команда:**

```powershell
# Переместить файл в архив
Move-Item "03-Resources/Knowledge/01_RULES/old_rule.md" `
          "03-Resources/Knowledge/04_ARCHIVES/2025/old_rules/"
```

---

### Шаг 3: Обновление ссылок

**В оригинальном файле (если остался):**

```markdown
> **Примечание:** Устаревшая версия перемещена в [`04_ARCHIVES/2025/old_rule.md`](04_ARCHIVES/2025/old_rule.md)
```

**В архивном файле:**

```markdown
> **Архив:** Этот файл устарел. Актуальная версия: [`01_RULES/new_rule.md`](../01_RULES/new_rule.md)
```

---

### Шаг 4: Запись в журнал

**Файл:** `reports/OPERATION_LOG.md`

```markdown
## ГГГГ-ММ-ДД ЧЧ:ММ Архивирование: old_rule.md

**Тип:** Перемещение в архив

**Путь:** `04_ARCHIVES/2025/old_rules/old_rule.md`

**Причина:** Заменён новой версией (new_rule.md)

**Статус:** ✅ Завершено

---
```

## 🗑️ УДАЛЕНИЕ ИЗ АРХИВА

**Критерии для удаления:**

- Материал полностью утратил историческую ценность
- Дубликат уже есть в архиве
- Материал повреждён

**Процесс:**

```markdown
[ ] 1. Проверить материал (ценность?)
[ ] 2. Проверить дубликаты
[ ] 3. Записать в OPERATION_LOG.md
[ ] 4. Удалить файл
[ ] 5. Git коммит (если крупное изменение)
```
