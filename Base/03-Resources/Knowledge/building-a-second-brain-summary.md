**Second Brain (Второй мозг)** — персональная цифровая система заметок, которая:
- ✅ Работает как внешний жесткий диск для мозга
- ✅ Имеет идеальную память (не забывает)

## 🧩 МЕТОД CODE (4 этапа)

```
┌─────────────────────────────────────────────────────────┐
│  CAPTURE → ORGANIZE → DISTILL → EXPRESS → (повторить)  │
└─────────────────────────────────────────────────────────┘
```

### **1️⃣ CAPTURE (Захват)**

**Принцип:** Сохраняйте только то, что **откликается** (resonates).

**Правило 10%:** Захватывайте не более **10%** от прочитанного/увиденного.

**Что захватывать:**

| Тип | Примеры |
|-----|---------|
| **Внешние активы** | Цитаты, закладки, конспекты встреч, изображения |
| **Внутренние активы** | Личные инсайты, истории, размышления, идеи |

**Критерии захвата:**
- ✅ **Вдохновляющее** — вызывает любопытство
- ✅ **Полезное** — имеет потенциал применения
- ✅ **Личное** — отражает ваш опыт
- ✅ **Удивительное** — бросает вызов пониманию

**Инструменты:**
- Notion, Evernote, Obsidian, Apple Notes
- Быстрый захват (виджет, горячие клавиши)
- **Захватывайте немедленно** — не доверяйте памяти

---

### **2️⃣ ORGANIZE (Организация)**

**Метод PARA** — организация по **действенности**, а не по темам.

```
📁 PARA/
├── 📁 01-Projects/     # Завершить (спринт)
├── 📁 02-Areas/        # Поддерживать (марафон)
├── 📁 03-Resources/    # Учиться (библиотека)
└── 📁 04-Archives/     # Сохранить (холодное хранилище)
```

| Категория | Вопрос | Пример |
|-----------|--------|--------|
| **Projects** | «Будет ли это завершено?» | «Запустить сайт к марту» |
| **Areas** | «Это пожизненная ответственность?» | «Здоровье», «Финансы» |
| **Resources** | «Я отвечаю или заинтересован?» | «Рецепты», «Маркетинг» |
| **Archives** | «Завершено или неактивно?» | «Налоги 2024», «Старые проекты» |

**Ключевое различие:**

| Projects | Areas |
|----------|-------|
| Есть дедлайн | Нет дедлайна |
| Цель: завершить | Цель: поддерживать |
| Пример: «Пробежать марафон» | Пример: «Фитнес» |

**Поток элементов:**
```
Inbox → PARA (Projects/Areas/Resources) → Active Work → Complete → Archives
```

### **3️⃣ DISTILL (Дистилляция)**

**Прогрессивное суммирование** — послойное выделение сути.

**4 слоя:**

| Слой | Название | Форматирование | Объём |
|------|----------|----------------|-------|
| **1** | Soil (почва) | Исходный текст | 100% |
| **2** | Oil (нефть) | **Жирный** | 10-20% |
| **3** | Gold (золото) | ==Хайлайт== | 10-20% |
| **4** | Gems (самоцветы) | 📝 Своими словами | 1-2% |

**Пример:**
```
Оригинал (500 слов):
«Информационная перегрузка — это повсеместная проблема. 
Герберт Саймон заметил: „Богатство информации создаёт бедность внимания". 
Чтобы бороться с этим, необходима структурированная система для цифровых заметок.»

После дистилляции:
«**Герберт Саймон: „Богатство информации создаёт бедность внимания"**. 
🟨 **Нужна структурированная система для заметок**.

💎 **Суть:** Без системы управления знаниями информация подавляет внимание.»
```

**Правило 10%:** На каждом слое сокращайте до 10-20% от предыдущего.

---

### **4️⃣ EXPRESS (Выражение)**

**Intermediate Packets (IP)** — переиспользуемые строительные блоки.

**5 типов IP:**

| Тип | Пример |
|-----|--------|
| **Distilled notes** | Конспект книги |
| **Outtakes** | Идеи, не вошедшие в прошлые проекты |
| **Work-in-process** | Документы, графики, планы |
| **Final deliverables** | Готовые результаты проектов |
| **Documents by others** | Материалы от коллег/клиентов |

**4 метода поиска IP:**
- 🔍 **Search** — когда знаете, что ищете
- 📂 **Browsing** — навигация по папкам
- 🏷️ **Tags** — кросс-категориальные связи
- 🎲 **Serendipity** — случайные открытия

**Пример создания контента:**
```
Проект: «Статья о продуктивности»
├── IP1: Заметка «Метод CODE» (из Resources)
├── IP2: Чек-лист «Еженедельный обзор»
├── IP3: Цитата Форте (из Capture)
└── IP4: Личный кейс применения (из Areas)
```

## 🎨 ТВОРЧЕСКИЙ ПРОЦЕСС

### **Divergence & Convergence**

| Фаза | Этапы CODE | Характер | Цель |
|------|------------|----------|------|
| **Divergence** | Capture + Organize | Спонтанно, хаотично | Генерация идей |
| **Convergence** | Distill + Express | Фокус, отбор | Создание результата |

### **Archipelago of Ideas (Архипелаг идей)**

**Метод:**
1. Соберите группу идей (divergence)
2. Свяжите их в логическом порядке (convergence)

**Преимущество:** Разделяет два сложных процесса:
- **Selection** (выбор идей) — требует открытого мышления
- **Sequencing** (упорядочивание) — требует фокуса
