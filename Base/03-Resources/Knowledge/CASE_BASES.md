## 1. 🎯 НАЗНАЧЕНИЕ

**Case-Based Reasoning (CBR)** — метод решения новых задач через адаптацию решений из прошлых похожих ситуаций.

**Основная идея:**
> "Если новая проблема похожа на старую — используй адаптированное решение"

## 3. 🔄 ПРОЦЕСС CBR

### 4 шага рассуждения:

```
1. ИЗВЛЕЧЬ (Retrieve)
   └─→ Новая проблема → Поиск похожих кейсов
   
2. ПЕРЕИСПОЛЬЗУЙ (Reuse)
   └─→ Адаптируй решение из кейса
   
3. АДАПТИРУЙ (Revise)
   └─→ Проверь, скорректируй под контекст
   
4. СОХРАНИ (Retain)
   └─→ Новый кейс → База прецедентов
```

## 4. 🔍 ПОИСК ПОХОЖИХ КЕЙСОВ

### Критерии相似ности:

| Критерий | Вес | Описание |
|----------|-----|----------|
| **Тип проблемы** | 40% | Ошибка UI / логика / производительность |
| **Контекст** | 25% | Проект, версия Unity, платформа |
| **Структура** | 20% | Архитектура решения |
| **Результат** | 15% | Успешность прошлого решения |

### Функция поиска:

```python
def find_similar_cases(new_problem, threshold=0.7):
    """
    Найти похожие кейсы в базе
    
    Args:
        new_problem: Описание новой проблемы
        threshold: Порог相似ности (0.0-1.0)
    
    Returns:
        List[case]: Отсортированные по相似ности кейсы
    """
    similar_cases = []
    
    for case in CASE_BASES:
        similarity = calculate_similarity(new_problem, case)
        if similarity >= threshold:
            similar_cases.append((case, similarity))
    
    return sorted(similar_cases, key=lambda x: x[1], reverse=True)
```

### CASE-001: Кнопка не отображается в Unity UI

**Дата:** 2 марта 2026 г.  
**Категория:** UI / Отображение

#### Проблема:
> Кнопка создана в коде, но не видна на сцене при запуске. Текст на кнопке тоже не отображается. Ошибок в консоли нет.

#### Решение:
```csharp
// 1. Проверить Canvas
var canvas = FindObjectOfType<Canvas>();
if (canvas == null) {
    canvas = new GameObject("Canvas").AddComponent<Canvas>();
}

// 2. Проверить RectTransform
var buttonObj = new GameObject("Button");
buttonObj.transform.SetParent(canvas.transform);
var rectTransform = buttonObj.AddComponent<RectTransform>();

// 3. Установить размеры
rectTransform.sizeDelta = new Vector2(160, 30);
rectTransform.anchoredPosition = Vector3.zero;

// 4. Добавить Image (фон кнопки)
var image = buttonObj.AddComponent<Image>();
image.color = Color.gray;

// 5. Добавить Button компонент
var button = buttonObj.AddComponent<Button>();

// 6. Добавить текст
var textObj = new GameObject("Text");
textObj.transform.SetParent(buttonObj.transform);
var text = textObj.AddComponent<Text>();
text.text = "Click Me";
text.rectTransform.sizeDelta = new Vector2(160, 30);
```

#### Результат:
✅ Кнопка отображается, текст виден, клик работает

#### Контекст:
- Проект: DragRaceUnity
- Unity: 6000.3.10f1
- Файлы: `MainMenuController.cs`, `MainMenu.unity`

#### Ключевые инсайты:
- Canvas должен быть на сцене
- Button требует Image для видимости фона
- RectTransform должен иметь корректные размеры
- Текст — отдельный дочерний объект

#### Применимость:
Использовать при создании UI элементов в Unity через код

### CASE-002: Ошибка кодировки UTF-8 в PowerShell скриптах

**Дата:** 2 марта 2026 г.  
**Категория:** Скрипты / Кодировка

#### Проблема:
> PowerShell скрипт не выполняется, ошибки с русскими символами в строках. Скрипт создавался в VS Code, но кодировка не указана.

#### Решение:
```powershell
# При создании файла явно указать кодировку
$content = "Привет, мир!"
[System.IO.File]::WriteAllText(
    "script.ps1", 
    $content, 
    [System.Text.UTF8Encoding]::new($false)  # UTF-8 без BOM
)

# Или через Set-Content
Set-Content -Path "script.ps1" -Value $content -Encoding UTF8
```

#### Результат:
✅ Скрипт выполняется корректно с русскими символами

#### Контекст:
- ОС: Windows 11
- PowerShell: 7.4
- VS Code: UTF-8 без BOM

#### Ключевые инсайты:
- PowerShell требует UTF-8 **без BOM**
- BOM вызывает ошибки парсинга
- VS Code по умолчанию может добавлять BOM

#### Применимость:
Все новые .ps1, .bat, .cmd скрипты

### CASE-003: Git коммитов накопилось много ahead of origin

**Дата:** 2 марта 2026 г.  
**Категория:** Git / Резервное копирование

#### Проблема:
> Локально 37 коммитов ahead of origin/master. Пользователь хочет загрузить на GitHub, но не уверен, нужно ли всё.

#### Решение:
```powershell
# 1. Проверить статус
git status
git log --oneline -10

# 2. Если всё важное — запушить
git push -u origin master

# 3. Если есть лишнее — сделать squash
git rebase -i HEAD~37  # Объединить в 1-2 коммита
git push -f -u origin master
```

#### Результат:
✅ Все коммиты загружены на GitHub

#### Контекст:
- Репозиторий: QwenPoekt
- Ветка: master
- Коммитов: 37 ahead

#### Ключевые инсайты:
- Коммитить локально — ежедневно (auto-commit-daily.ps1)
- PUSH — только крупные изменения
- 37 ahead — нормально для локальной разработки

#### Применимость:
Ежедневная работа с Git

### Когда использовать CBR:

✅ **Типовые задачи:**
- Создание UI элементов
- Настройка Git
- Ошибки кодировки

- "Кнопка не работает" → CASE-001

❌ **Не использовать:**
- Уникальные задачи (нет аналогов)
- Критичные изменения (требуется полный анализ)
