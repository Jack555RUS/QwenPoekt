## 🔍 ТРЕБОВАНИЯ К АНАЛИЗУ

### 1. Извлечение ключевых идей

- Выдели 3-5 ключевых идей
- Каждая идея — 1-2 предложения
- Используй русский язык

### 2. Оценка ценности (1-5)

| Оценка | Критерий |
|--------|----------|
| **5** | Критично важно для Ядра |
| **4** | Очень полезно для Resources |
| **3** | Полезно, но не срочно |
| **2** | Малоценно, архив |
| **1** | Бесполезно, удалить |

### 3. Рекомендации по категоризации (PARA)

- **Projects:** Если есть дедлайн
- **Areas:** Если постоянная ответственность
- **Resources:** Если библиотека знаний
- **Archives:** Если устарело/завершено

### 4. Метаданные

- **title:** Название на русском
- **tags:** 3-5 тегов (kebab-case)
- **related:** Связанные файлы в проекте

### 5. Применение в проекте

- Конкретные действия
- Файлы для создания/изменения
- Скрипты для добавления

---

# 📊 АНАЛИЗ: Markdown Best Practices 2025

## 🔑 КЛЮЧЕВЫЕ ИДЕИ

1. **kebab-case для имён файлов** — нижний регистр, дефисы, без пробелов
2. **Front matter обязателен** — title, version, date, status, tags
3. **Даты в формате ISO 8601** — YYYY-MM-DD для хронологической сортировки
4. **Длина имени ≤50 символов** — достаточно информативно, но кратко
5. **Автоматизация проверок** — CI/CD для документации

## 💎 САМАЯ СУТЬ

Используйте единый стандарт именования и метаданные для масштабируемой документации.

## 📝 КРАТКОЕ ИЗЛОЖЕНИЕ

Статья описывает лучшие практики создания Markdown-документации в 2025 году. 
Основные рекомендации: kebab-case для имён файлов, front matter для метаданных, 
ISO 8601 для дат, автоматизация проверок через CI/CD. Исследование показало 
улучшение навигации на 23% при использовании единого стандарта.

### **Скрипт для пакетного анализа:**

```powershell
# scripts/batch-analyze-materials.ps1
param(
    [string]$InputPath,
    [string]$OutputPath
)

$materials = Get-ChildItem $InputPath -Filter "*.txt"

foreach ($material in $materials) {
    $content = Get-Content $material -Raw
    
    # Отправить ИИ на анализ
    $analysis = Invoke-AIAnalysis -Prompt $content
    
    # Сохранить результат
    $analysis | Out-File "$OutputPath\$($material.BaseName)-analysis.md"
}
```

## 💡 СОВЕТЫ

### **Для лучших результатов:**

1. **Вставляйте полный текст** — чем больше контекста, тем лучше анализ
2. **Указывайте тип материала** — книга, статья, код, идея
3. **Проверяйте рекомендации** — ИИ может ошибаться в путях файлов
4. **Сохраняйте анализы** — в `_drafts/analyses/` для истории

---

### **Частые ошибки:**

| Ошибка | Решение |
|--------|---------|
| **Слишком кратко** | Вставляйте больше контекста |
| **Нет источника** | Всегда указывайте URL/книгу |
| **Игнорирование оценки** | Оценка 1-2 = удалить/архив |
| **Нет тегов** | Теги обязательны для поиска |
