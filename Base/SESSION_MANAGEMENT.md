# 🔄 УПРАВЛЕНИЕ СЕССИЯМИ — Полное Руководство

**Версия:** 1.0
**Дата:** 2026-03-04
**Статус:** ✅ Активно

---

## 📋 НАЗНАЧЕНИЕ

Это руководство описывает правильный процесс:
1. ✅ **Завершения сессии** (сохранение контекста)
2. ✅ **Возобновления сессии** (восстановление контекста)
3. ✅ **Автосохранения** (каждые 2 минуты)

---

## 🚀 БЫСТРЫЙ СТАРТ

### Завершение сессии (30 секунд):

```powershell
# Автоматическое завершение
.\scripts\end-session.ps1 -Reason "Задача выполнена"

# Или вручную (триггеры в чате):
"выход", "перезагрузка", "выключение", "stop", "break"
```

### Возобновление сессии (1 команда):

```powershell
# Автоматическое восстановление
.\scripts\start-session.ps1 -Resume

# Или команда в чате:
/resume
```

## 📚 ПОДРОБНЫЙ ПРОЦЕСС

### 1️⃣ ЗАВЕРШЕНИЕ СЕССИИ

#### Автоматическое завершение:

```powershell
.\scripts\end-session.ps1 -Reason "Причина завершения"
```

**Параметры:**
| Параметр | Описание | Пример |
|----------|----------|--------|
| `-Reason` | Причина завершения | `"Задача выполнена"` |
| `-NoCommit` | Не делать коммит | `-NoCommit` |
| `-Verbose` | Подробный вывод | `-Verbose` |

**Что создаётся:**
- ✅ `reports/SESSION_HANDOVER.md` — контекст сессии
- ✅ `.resume_marker.json` обновляется
- ✅ Git коммит (если есть изменения)

---

#### Ручное завершение (триггеры в чате):

**Слова-триггеры:**
```
"выход", "exit", "quit", "bye"
"перезагрузка", "reboot", "restart"
"выключение", "shutdown"
"конец сессии", "завершаю"
"до завтра", "goodbye"
```

**Что происходит:**
1. ИИ обнаруживает триггер
2. MCP Session Saver сохраняет сессию
3. Создаётся `SESSION_HANDOVER.md`
4. Обновляется `.resume_marker.json`

---

#### Чек-лист перед завершением:


## 🔄 АВТОСОХРАНЕНИЕ

### Настройка:

**Файл:** `.qwen/session-rules.json`

```json
{
  "session": {
    "auto_save": {
      "enabled": true,
      "interval_minutes": 2,
      "format": "hybrid",
      "path": "sessions/"
    }
  }
}
```

### Как работает:

```
Каждые 2 минуты:
  1. MCP Session Saver читает диалог
  2. Сохраняет в sessions/YYYY-MM-DD_HH-mm/
  3. Обновляет .resume_marker.json
```

**Структура сессии:**
```
sessions/2026-03-04_00-30/
├── chat.md           # Переписка
├── metadata.json     # Метаданные
└── commands.log      # Команды PowerShell
```

## ⚠️ ТИПИЧНЫЕ ПРОБЛЕМЫ

### Проблема 1: Нет маркера восстановления

**Симптом:**
```
❌ Маркер восстановления не найден
```

**Решение:**
```powershell
# Проверить sessions/
Get-ChildItem sessions\ -Directory | Sort-Object CreationTime -Descending

# Вручную прочитать последнюю сессию
Get-Content sessions\2026-03-03_17-00\chat.md
```

---

### Проблема 2: SESSION_HANDOVER.md не найден

**Симптом:**
```
❌ SESSION_HANDOVER.md не найден
```

**Решение:**
```powershell
# Создать вручную
New-Item reports\SESSION_HANDOVER.md -ItemType File

# Заполнить шаблоном
@'
# SESSION HANDOVER

**Дата:** 2026-03-04
**Задача:** [Задача]
**След. шаг:** [Шаг]
'@ | Out-File reports\SESSION_HANDOVER.md -Encoding UTF8
```

---

### Проблема 3: Git не закоммичен

**Симптом:**
```
⚠️ Найдено 5 незакоммиченных изменений
```

**Решение:**
```powershell
# Закоммитить перед завершением
git add .
git commit -m "Backup: Перед завершением сессии"

# Или отложить изменения
git stash
```
